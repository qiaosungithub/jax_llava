Four-Trunk VLM: 第二次看图的 5 种架构设计

面向当前 PaliGemma-baseline 代码的架构 brainstorm report。核心问题：Trunk 3 应该怎样同时利用 prompt、第一次图像特征和原始 pixel/patch 信息。

结论先行。 我不建议把第二次看图只做成 prompt query -> img_features_1。这样虽然接近 DETR 的 query/object-slot 思路，但会被第一次 1D 压缩的信息瓶颈限制。更有研究价值的 Trunk 3 应该至少同时看到两类视觉信号： img_features_1 作为第一次抽象语义记忆，patch_tokens / pixels 作为可重读的低层或空间细节。

1. 现有代码约束

当前实现不是普通 SigLIP prepend，而是一个适合做 4-trunk 的 PrefixMAE-style encoder-decoder：

  • models/siglip_enc_dec.py 中的 ImageEncoder 结构是：patch embedding -> patch self-attention -> learnable 1D tokens cross-attend patch tokens -> token self-attention -> K 个 abstract tokens。
  • models/paligemma_enc_dec.py 中的 PaliGemmaEncDec 把 image tokens project 到 LLM hidden size，然后 prepend 到 text embeddings，frozen LLM 负责 next-token loss。
  • 代码已经有 txt_feature_layer 这个方向：可以先让文本通过低层 LLM 得到更丰富的 prompt_features，再和图像特征组合。
  • 现有 CrossAttentionBlock 已经足够实现多数 Trunk-3 refiner；最大缺口是让 image encoder 暴露中间 patch_tokens。
Trunk 1: problem-blind image
Trunk 2: prompt
Trunk 3: prompt-aware image
Trunk 4: answer
Trunk 1 Image first pass pixels -> patch_tokens -> img_features_1 problem-blind, trainable vision path
Trunk 2 Prompt encoder prompt_ids -> prompt_features raw embeddings or lower frozen LLM layers
Trunk 3 Image second pass pixels + img1 + prompt -> img_features_2 the design target of this report
Trunk 4 Answer decoder [img1 | prompt | img2 | answer] LM loss only on answer tokens

2. 设计目标

一个好的 Trunk 3 需要满足四个条件：

  1. Prompt-aware: 输出特征必须显式依赖问题，而不是只依赖图像。
  2. Pixel-aware: 不能只在第一次压缩后的 img_features_1 上做检索，否则无法恢复被压缩掉的 OCR、小物体、局部属性。
  3. 兼容 frozen LLM supervisor: 主要监督仍来自 answer NTP loss，LLM 尽量冻结。
  4. 可做干净 ablation: 能和 3-trunk、4-trunk alpha (img_features_2 = img_features_1)、以及不同 pixel access 版本比较。

3. 五种本质不同的 Trunk-3 架构

方案 1: Prompt-conditioned Pixel Q-Former / DETR-style Slots

推荐 first implementation DETR-style queries uses patch tokens moderate cost

核心想法。 用一组 learnable slots 或从 img_features_1 初始化的 slots 作为 query。每层让 slots 同时访问 prompt、第一次图像特征和 patch-level pixel features。

Trunk 1 exposes:
  patch_tokens      # [B, N_patch, Dv], from pixel/patch stream
  img_features_1   # [B, K, Dlm], first abstract image memory

Trunk 2 exposes:
  prompt_features  # [B, Lp, Dlm]

Trunk 3:
  q = learned_slots or project(img_features_1)
  for block in QFormer2:
      q = SelfAttn(q)
      q = CrossAttn(q, prompt_features)
      q = CrossAttn(q, patch_tokens)
      q = CrossAttn(q, img_features_1)
      q = FFN(q)
  img_features_2 = project(q)

为什么它解决问题。 DETR-style cross-attention 可以让 prompt-conditioned slots 学会“该看哪些区域”。但这里不是只 attend 到 img_features_1，而是把 patch_tokens 也作为 K/V，所以第二次看图确实能重新访问 pixel-derived spatial evidence。

当前代码落地。 最小改动是给 ImageEncoder.__call__ 增加 return_intermediates，返回 Stage-1 patch self-attn 后的 x。然后新增一个 PromptAwareQFormer，复用现有 CrossAttentionBlock 思路。

风险。 如果 slots 只从 learnable query 开始，早期训练可能难；可以把 query 初始化为 stop_grad(img_features_1) 的线性投影，或用 learned_slots + prompt_pooled_gate * img_features_1。

方案 2: MMDiT-style Multi-stream Refiner

strongest fusion multi-stream higher cost full model candidate

核心想法。 把三类输入当成三个 modality stream：pixel/patch stream、第一次 image feature stream、prompt stream。每层既保留 modality-specific projection，也允许 shared attention 混合。

patch_stream  = patch_tokens + patch_pos + type_pixel
img1_stream   = img_features_1 + abstract_pos + type_img1
prompt_stream = prompt_features + text_pos + type_prompt

for block in MultiStreamRefiner:
    patch_stream, img1_stream, prompt_stream =
        MultiModalAttention(patch_stream, img1_stream, prompt_stream)

img_features_2 = AttentionalPooler(
    queries=learned_or_img1_queries,
    keys_values=[patch_stream, img1_stream, prompt_stream],
)

本质区别。 方案 1 是 slots 主动读取 context；方案 2 是 patch tokens 本身也被 prompt 改写。它更接近 MMDiT / deep fusion 的精神：prompt 不是外部 memory，而是参与 image representation 的逐层构造。

优点。 对 OCR、空间关系、细粒度属性可能更强，因为 dense patch grid 在多层中直接吸收问题信息。

缺点。 复杂度明显更高，尤其当 N_patch 很大时。实现上也会偏离当前 PrefixMAE 的简洁结构。建议等方案 1 证明 Trunk 3 有收益后再做。

方案 3: SAM-style Two-way Prompt-Image Refiner

two-way attention dense image map interpretable masks good for localization

核心想法。 参考 SAM 的 promptable decoder：prompt tokens attend image tokens，image tokens 也反向 attend prompt tokens。最后从被 prompt 改写过的 dense image map 中 pool 出 img_features_2。

dense_img = patch_tokens reshaped as 2D grid
prompt_q  = prompt_features or prompt_summary_tokens

for block in TwoWayRefiner:
    prompt_q  = SelfAttn(prompt_q)
    prompt_q  = CrossAttn(prompt_q, dense_img)
    dense_img = CrossAttn(dense_img, prompt_q)
    prompt_q, dense_img = FFN(...)

optional:
  region_mask = MaskHead(dense_img, prompt_q)

img_features_2 = AttentionalPooler(queries, dense_img + prompt_q)

为什么不同。 它不是把 prompt 当成普通 K/V，也不是把所有 modality 合成一个大 transformer，而是有明确的双向交互结构。prompt token 负责提出需求，dense image token 负责被需求改写。

额外收益。 可以输出 attention map 或 soft region mask，作为 debug artifact：看模型第二次到底在看哪里。对 RefCOCO、TextVQA、小物体属性类任务很有诊断价值。

风险。 如果最终任务只需要全局语义，two-way dense refiner 可能比 Q-Former 更贵但收益不明显。最好和 localization/OCR-heavy eval 一起看。

方案 4: Prompt-adapted Second ViT / Second PrefixMAE

true second visual pass unshared or partially shared weights highest semantic clarity expensive

核心想法。 第二次看图不是在第一次特征上做 refiner，而是重新跑一个视觉 encoder。这个 encoder 可以与第一次共享 patch embedding / lower blocks，也可以完全独立。prompt 通过 cross-attention、AdaLN/FiLM、或 prefix tokens 注入。

Trunk 3 second encoder:
  x = PatchEmbed2(pixels)
  memory = [project(img_features_1), prompt_features]

  for layer in SecondViT:
      x = SelfAttn2D(x)
      x = CrossAttn(x, memory)        # every N layers or every layer
      x = AdaLN(x, prompt_summary)    # optional
      x = FFN(x)

  img_features_2 = QFormer_or_pooler(x)

适用场景。 如果研究问题是“第二次看图的 weights 是否应该不 share”，这是最直接的实验。它可以回答：prompt-aware visual computation 是否需要从 pixel encoder 的早期层就开始发生。

参数共享建议。 可以从三档做 ablation：只共享 patch embed；共享前半 ViT；完全不共享。完全不共享最纯粹，但数据量要求最大。

风险。 训练成本高，且可能把收益混在“更多参数/更多 compute”里。需要和同等参数量的 deeper single-pass baseline 比较。

方案 5: Active High-res Glimpse / Crop Re-encoder

active perception pixel crop access OCR and small objects routing risk

核心想法。 第一次看图负责产生全局理解；prompt + img_features_1 预测若干 region/glimpse；第二次从原图裁剪这些区域，用小 encoder 或共享 encoder 高分辨率重看。

region_queries = DETRDecoderQueries(prompt_features, img_features_1)
boxes_or_masks = RegionHead(region_queries)

glimpses = ROIAlign_or_crop_resize(pixels, boxes_or_masks)
glimpse_tokens = SmallVisionEncoder(glimpses)

img_features_2 = aggregate(
    region_queries,
    glimpse_tokens,
    prompt_features,
)

为什么值得考虑。 很多 VQA 错误不是因为全图语义不够，而是因为需要重看局部：牌子上的字、表格单元、小物体颜色、远处计数。这个方案把 Trunk 3 变成一个主动视觉系统。

训练策略。 第一版可以不用 hard crop，先做 differentiable soft attention over patch grid；稳定后再接 ROIAlign/crop。也可以用 auxiliary reconstruction/OCR/localization loss 稳定 region selection。

风险。 如果 region selector 早期学坏，后面的 crop encoder 没有梯度信号。需要 warmup：先允许全图 patch attention，再逐渐收紧到 top-k regions。

5. 推荐实验顺序

第一阶段：做最小可验证版本。 先实现方案 1，并保留两个低成本 control： 3-trunk baseline、 4-trunk alpha: img_features_2 = img_features_1、 4-trunk Pixel Q-Former: img_features_2 = f(patch_tokens, img_features_1, prompt_features)。
  1. Expose intermediates: 给 PrefixMAE.encode 增加可选返回 patch_tokens。优先返回 patch self-attention 后的 tokens，而不是 raw patch embedding。
  2. Add Q-Former2: 新增 PromptAwareQFormer，输入 patch_tokens、img_features_1、prompt_features，输出 K2 个 LM-space tokens。
  3. Build 4-trunk mask: sequence 改为 [img1 | prompt | img2 | answer]。前三个 trunk 内部 bidirectional，trunk 之间 block-causal，answer 内部 causal。
  4. Run ablations: 比较 no-pixel Q-Former、with-patch Q-Former、with-patch+img1 memory。这样可以直接回答“pixel access 到底有没有价值”。
  5. Only then scale: 如果方案 1 有收益，再试方案 3 或方案 2；如果收益主要在 OCR/局部任务上，再试方案 5。

6. 关键 ablation

Ablation 目的 预期解释
3-trunk vs 4-trunk alpha 测试“同一图像 token 插两次”是否本身有用 如果 alpha 赢，说明 LLM 内部 attention 已经能把第二份 image tokens prompt-aware 化
QFormer2(img1, prompt) vs QFormer2(patch, img1, prompt) 测试 pixel/patch reread 是否必要 如果后者赢，说明第一次 1D 压缩确实丢掉了问题相关细节
raw prompt_embeds vs lower-LLM prompt_features 测试 Trunk 2 是否需要真正的 prompt encoder 如果 prompt_features 赢，说明 frozen LLM 的中间表示对视觉重读有监督价值
share QFormer1 queries vs separate QFormer2 queries 测试两次看图是否应共享 visual slot vocabulary shared 更稳，separate 更灵活；结果能指导参数共享设计
patch tokens from early layer vs late layer 测试第二次看图需要低层纹理还是高层语义 OCR/小物体可能偏 early，普通 VQA 可能偏 late

7. 我的优先级判断

如果目标是尽快产出有说服力的实验，我会按这个顺序推进：

  1. 先做方案 1。 它最贴合现有 PrefixMAE，改动有限，同时正面回答 pixel access 问题。
  2. 保留方案 3 作为第二个强 baseline。 如果希望在报告中讲“第二次看图更像 promptable segmentation / localization”，SAM-style two-way refiner 更容易解释和可视化。
  3. 方案 4 用作上界实验。 它成本高，但能回答 unshared second visual encoder 是否必要。
  4. 方案 2 和 5 暂不作为第一版。 方案 2 是 full fusion model，方案 5 是 active perception model；都很有潜力，但第一轮会让变量太多。
最需要避免的坑。 不要只做 prompt -> img_features_1 的 cross-attention 后就声称“第二次看图”。那更像是在第一次视觉摘要上做 query，不是真正 reread image。除非它只是 no-pixel ablation，否则主实验应包含 patch_tokens 或 crop/pixel path。
