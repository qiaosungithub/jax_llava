PaliGemma-style Visual Understanding Baseline

This report presents an autoregressive multimodal baseline built on a Vision Encoder + Gemma Decoder. It consolidates architecture, implementation details, training settings, evaluation protocols, and current experimental results.

Architecture

Model Overview

  • Overall paradigm: image and text are combined and decoded with next-token prediction (AR generation).
  • Vision encoder: flexible transformer-based module; the current implementation uses a 1-D tokenizer route.
  • Language decoder: Gemma initialized from pretrained checkpoint; the vision side is trained from scratch.

Tensor Flow

  • image (B,H,W,C) -> patchify (B,T,C) -> vision encoder (B,T,C)
  • 2-layer MLP + RMSNorm -> image embeddings (B,T,D)
  • text -> Gemma tokenizer -> text embeddings (B,T,D)
  • concat(image, text) -> Gemma decoder -> logits (B,T,V) -> cross entropy

Attention Mask (same policy as PaliGemma)

  • Image tokens and prompt tokens use full attention.
  • Answer tokens use causal attention.
1-D Encoder

Current Image Encoder Implementation

  • From models/siglip_enc_dec.py: three-stage encoding with patch self-attn, learnable-token cross-attn, then token self-attn.
  • Default MAE-B style setup: enc_hidden_dim=768, num_learnable_tokens=256, and stage depths 3/6/3.
  • Optional mid-stream visibility masking (enc_cross_attn_split) controls information bottleneck in visual token flow.

Generation Strategy

  • Default decoding is greedy (beam_size=1).
  • Beam search is implemented, but currently provides limited gains for QA-style tasks.

LM Backbones (present in this codebase)

  • Base: gemma3_270M (main baseline).
  • Larger: gemma2_2B (supported in loader and training path).

For the referenced run, the log reports 358,511,104 trainable parameters (~358.5M).

<loc> Token Mechanism

  • Bounding-box coordinates are quantized into [0, 1023] and represented by four discrete tokens (ymin/xmin/ymax/xmax).
  • Tokens are formatted as <loc0000> ... <loc1023>; embeddings are initialized at training startup.
  • Initialization follows the sinusoidal scheme described in your note, and these embeddings remain learnable.

Finetuning Data

  • Two practical finetuning routes are used: (1) finetune on VQAv2 train / TextVQA train for matched evaluation; (2) alternative SFT on LLaVA-1.5-Instruct then broader zero-shot evaluation.
  • The current example in configs/finetune_config.yml uses llava-1.5.

Input Resolution

  • Pretraining commonly uses fixed resolution without a separate high-resolution stage.
  • With large-patch transformers, both 256 and 512 are practical options.
Training

Training Setup (Pretrain / Finetune)

Pretrain (from remote_run_config)

  • Optimizer: Muon in the main setup, with lr=2e-4, wd=0.05; Adam is available as an alternative.
  • Schedule: lr_schedule=const, warmup_steps=30000.
  • Scale: batch_size=1024, num_steps=300000.
  • Soft caps: attn_logits_soft_cap=50.0, final_logit_softcap=30.0.
  • Vision path: siglip_from_scratch=True with MAE-B style encoder settings.

Finetune (from finetune_config)

  • Optimizer: Muon, typically lr=2e-5, wd=0.02.
  • Schedule: lr_schedule=cos, warmup_steps=4000, num_steps=40000.
  • Scale: batch_size=256.
  • Soft caps are disabled in the shown finetune file (both set to 0.0).

CFG-related Training Terms

  • vlm_loss_weight: standard image-conditioned NTP loss.
  • text_only_loss_weight: text-only branch loss (corresponding to loss_text_only).
  • cfg_loss_weight: CFG-style contrastive term (corresponding to loss_cfg).
  • alpha: CFG contrastive strength; alpha > 0 enables text-only forward pass.
Evaluation

Benchmarks and Protocols

  • VQAv2: train/val available; standard accuracy tracking.
  • MME: report both MME-P and MME-S scores.
  • TextVQA: train/val available; currently a key weakness area.
  • POPE: can be run zero-shot; report macro F1 and split-wise metrics (random/popular/adversarial).
  • ImageNet KNN: supports online partial monitoring (128 images/class) and final full evaluation; can be run zero-shot.
  • MMBench: treated as non-zero-shot in your current setup notes.
