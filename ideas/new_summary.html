PaliGemma-style Visual Understanding Baseline (Updated Consolidated Summary)

This report consolidates the latest architecture/runtime changes, data expansion, evaluation upgrades, and current operating conventions. It is intended as a handoff-quality update for collaborators.

System Snapshot

Current Baseline Direction

  • Core model remains a Vision Encoder + Gemma Decoder autoregressive VLM.
  • Training now supports both full-parameter updates and selective LM freezing via config switch.
  • Prompting has moved from task-prefix tokens to natural-language prompts as the default policy.
  • Evaluation orchestration is now task-list driven from training.online_eval_tasks and training.final_eval_tasks.
Architecture and Optimization

Model and Train-Step Updates

Encoder/Decoder Baseline

  • Vision side continues with MAE-B style dimensions and a learnable-token bottleneck route in siglip_enc_dec.
  • Language side remains Gemma-based (mainly gemma3_270M in active runs).

LM Freeze Path

  • New train-step logic supports splitting trainable vs frozen parameter trees and merging updates safely.
  • Gradient flow is constrained to active trainable branches while optimizer state compatibility is preserved.
  • Useful for ablations and stability checks during multimodal adaptation.

Prompting Policy

Migration to Natural Prompts

  • Legacy prefixes like caption en, answer en, ocr en, and detect en are removed from active train/eval paths.
  • QA-style prompting now uses direct question forms (with a trailing ? when missing) plus newline.
  • Grounding prompts are explicit natural instructions (for example, “Locate the region described by this phrase ...”).
  • Dataset-specific prompt templates remain, but all are human-readable and prefix-free.

Motivation

  • Reduce over-reliance on handcrafted task tags and align train/eval instructions with actual language use.
  • Enable cleaner transfer across mixed datasets without token-level task gating artifacts.
Evaluation Stack

Task Coverage and Runtime Interface

Supported Eval Tasks (Current)

  • VQAv2, MME, TextVQA, POPE, MMBench, ImageNet KNN, and RefCOCOg.
  • MMBench and POPE now inherit prefix-free defaults from config and evaluator-level prompt builders.

Scheduling Interface

  • Online eval cadence uses training.online_eval_per_step + training.online_eval_tasks.
  • Terminal/summary eval uses training.final_eval_tasks.
  • Configs were updated to remove old per-task *_per_step fields and deprecated eval.eval_* toggles.
Data and Operations

Infra and Reliability Improvements

  • Added region/zone-aware uploader tooling to minimize cross-region friction in TPU workflows.
  • COCO upload now uses tar shards to avoid tiny-file API overhead and shell argument-length failures.
  • RefCOCOg val annotations were stabilized via local SSD path usage for evaluation jobs.
  • Upload scripts include fallback and sanitization logic to handle brittle third-party source behavior.
