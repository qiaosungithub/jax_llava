Project Update Report (Recent Additions)

This document focuses on newly implemented components since the previous baseline summary, including new tasks, evaluation infrastructure, data pipelines, and prompt-system migration.

New Evaluation Task

RefCOCOg Evaluation Added End-to-End

  • Implemented a new evaluator in evals/eval_refcocog.py for referring-expression grounding.
  • The evaluator decodes model outputs into quantized <loc####> tokens, recovers predicted boxes, and computes IoU-based metrics.
  • Main reported metrics include Acc@IoU thresholds and mIoU, with host/process-safe aggregation for distributed runs.
  • Task registration added in evals/eval.py so refcocog can be triggered from unified eval task lists.
  • Prompt switched to natural language: Locate the region described by this phrase: {phrase} (prefix-free).
New Pretrain Data

Detection and TextCaps Data Paths Integrated

Visual Genome Detection (genome_det)

  • Added a new pretraining route in input_pipeline.py that formats examples as phrase-to-box prediction.
  • Integrated a dedicated iterable dataset path for region/object supervision with shuffle-buffer sampling.
  • Target format uses quantized bbox location tokens, fully aligned with the model's existing <loc####> vocabulary.

TextCaps

  • Added dataset mapping and routing for textcaps-train and textcaps-val in utils/data_util.py.
  • Added textcaps-specific prompt behavior in input_pipeline.py to keep outputs concise and text-reading aware.
  • Upload pipeline now supports WebDataset tar shard generation where each image is stored once and captions are grouped.
Prompt Migration

Task Prefix Removal and Natural Prompt Standardization

  • Removed task-token prefixes in training and eval paths (for example caption en, answer en, ocr en, detect en).
  • QA/OCR-style tasks now use direct natural prompts, typically {question}? followed by newline.
  • Applied to eval implementations including eval_vqav2.py, eval_textvqa.py, eval_mme.py, eval_refcocog.py, eval_pope.py, and eval_mmbench.py.
  • Config defaults updated to match runtime behavior (for example eval.pope_prompt_template and eval.mmbench_prompt_prefix).
Training Runtime

Online/Final Evaluation Interface Updated

  • train.py now uses task-list based evaluation scheduling via training.online_eval_per_step, training.online_eval_tasks, and training.final_eval_tasks.
  • Legacy per-task step keys were replaced in configs/remote_run_config.yml and configs/finetune_config.yml.
  • The new scheme makes eval orchestration explicit, centralized, and easier to maintain as tasks expand.
Optimization

LM Freeze Training Option Added

  • Added trainable/frozen parameter split-merge support (via utils/frozen_util.py and train.py integration).
  • When training.freeze_lm is enabled, gradients are computed only on trainable subtrees while preserving optimizer state structure.
  • This supports controlled ablations and efficient adaptation experiments with reduced trainable footprint.
Data Infra

Uploader and Dataset Logistics Improvements

  • Added zone-aware upload scripts for RefCOCOg, COCO, and TextCaps in the companion tooling directory.
  • Switched COCO upload from many-small-file mode to tar-sharded mode to avoid API overhead and argument-length failure modes.
  • RefCOCOg validation annotations are now resolved from local SSD path by default for stable access in eval jobs.

Status

Net Effect of the Update Batch

  • Task coverage expanded: grounded referring-expression evaluation and detection-style pretraining are now first-class.
  • Prompting policy unified: old prefixed prompt conventions are replaced by natural-language prompting across train/eval.
  • Ops reliability improved: data upload and path resolution are more robust for multi-region TPU workflows.
