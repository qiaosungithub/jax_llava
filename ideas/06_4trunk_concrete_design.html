4-Trunk VLM — Concrete Design (LLaVA, Flamingo, and Cambrian per-block variants)

Generalizing the three 3-trunk variants from brainstorm/05 to the 4-trunk setting (Image · Prompt · Image-again · Answer)

Context for this document. Brainstorm/05 built three concrete variants of the 3-trunk VLM: C (LLaVA-style: vision tokens prepended once at the LLM input), A (Flamingo-style: gated cross-attention into the upper LLM layers), and B (Cambrian-style per-block: a Q-Former at every decoder layer). This document extends each of those variants to the 4-trunk setting: the image is "looked at twice" — first problem-blind, then again after the question has been read. The four trunks, in order, are: (1) Vision Encoder on the image, problem-blind; (2) Prompt Encoder for the question; (3) Vision Re-encoding of the same image, now able to see the prompt; (4) Answer Decoder, autoregressive. The trunks are causally ordered: each trunk can attend to all previous trunks' outputs, but not future ones.
Running example used throughout.
  • Image: a 224×224 RGB photo of a black cat sitting on a chair (same as 3-trunk doc)
  • Prompt: "What color is the cat?" → 6 tokens
  • Target answer: "The cat is black." → 5 tokens

Intuition for the 4th trunk. Trunk 1 sees the cat-on-chair holistically and may emphasize many features (chair, lighting, background). After the question is read in Trunk 2, Trunk 3 re-encodes the same pixels but now "knows it should look at the cat's color" — its output should be more focused on cat-color-relevant features. Trunk 4 then generates the answer using both views (the broad first look + the focused second look) plus the prompt.

1. Overall 4-trunk architecture

Figure 1: Overall 4-trunk architecture. Trunks 1 and 3 both consume the image; Trunk 3 also reads Trunks 1 and 2 (i.e., it is "prompt-aware"). Trunk 4 sees everything before. The variant (C/A/B) governs how Trunk 4 (and to a lesser extent Trunk 2) is realized inside the LLM. Two mechanisms for Trunk 3: α (zero-cost — vis_tokens_2 = vis_tokens_1, prompt-awareness emerges purely from the LLM's attention pattern) or β (a small trained vision refiner with cross-attention to prompt features).

2. Trunk 3 — the two re-encoding mechanisms

Before tackling each variant, define the two ways Trunk 3 can produce vis_tokens_2:

Mechanism α — positional only (zero-cost)

Set vis_tokens_2 = vis_tokens_1 at the input to the LLM. The two visual encodings are identical at the encoder boundary; they only diverge once they are inside the LLM, because they sit at different positions in the sequence and are subject to different attention masks (vis_1 sees nothing else; vis_2 sees vis_1 and the prompt). After one self-attention layer they have already started diverging — this is the same trick that lets a transformer LM produce different next-token predictions at different positions even though the early embeddings are the same per-token-id. Trainable params added by Trunk 3: zero. This is the natural Trunk 3 for Variant C.

Mechanism β — vision refiner (trained, ~30–60M params)

A small transformer stack — the "vision refiner" — sits on top of Trunk 1's output and produces a genuinely different feature sequence by mixing in prompt information. Architecture: L_refiner = 4 blocks, each with self-attention over vis_tokens_1, cross-attention to prompt_features (or prompt_embeds in Variant C), and an FFN. Output shape: same as vis_tokens_1, namely [196, 4096].

Figure 2: Trunk 3 vision refiner (Mechanism β). A 4-block transformer stack that re-mixes vis_tokens_1 using prompt_features as cross-attention K/V. Output shape is identical to the input, so downstream code only needs to know "vis_tokens has shape [196, 4096]" and not which trunk it came from.

Pseudocode for Mechanism β

class VisionRefiner(nn.Module):
    """Small transformer that re-mixes vis_tokens_1 conditioned on prompt features."""
    def __init__(self, L_refiner=4, d=4096, n_heads=32):
        super().__init__()
        self.blocks = nn.ModuleList([
            VisionRefinerBlock(d=d, n_heads=n_heads) for _ in range(L_refiner)
        ])

    def forward(self, vis_tokens_1, prompt_features):
        h = vis_tokens_1                                      # [B, 196, 4096]
        for block in self.blocks:
            h = block(h, prompt_features)
        return h                                              # [B, 196, 4096]


class VisionRefinerBlock(nn.Module):
    def __init__(self, d, n_heads):
        super().__init__()
        self.self_attn  = SelfAttention(d, n_heads)
        self.cross_attn = CrossAttention(d, kv_d=d, n_heads=n_heads)
        self.ffn = SwiGLU(d, 4 * d)
        self.norm1 = nn.LayerNorm(d)
        self.norm2 = nn.LayerNorm(d)
        self.norm3 = nn.LayerNorm(d)

    def forward(self, vis, prompt):
        vis = vis + self.self_attn(self.norm1(vis))                 # bidirectional self-attn over vis
        vis = vis + self.cross_attn(self.norm2(vis), kv=prompt)     # vis attends to prompt
        vis = vis + self.ffn(self.norm3(vis))
        return vis

3. Variant C 4-trunk — LLaVA-style (the simplest)

Pure LLaVA spirit: keep everything at the LLM input level, no cross-attention, no per-block injection. Use Mechanism α for Trunk 3. The LLM's own self-attention is what makes Trunk 3 useful — vis_tokens_2 sits after the prompt in the sequence and so attends to the prompt; vis_tokens_1 sits before the prompt and does not.

Figure 3: Variant C 4-trunk. The four trunks are concatenated into a single LLM input sequence in causal order. The attention mask (right) enforces the trunk ordering: vis_tokens_1 attends only to itself; the prompt attends to itself and to vis_tokens_1; vis_tokens_2 attends to all of vis_1, p, v_2; the answer is fully causal and attends to everything. Trunk 3 is "free" in this variant — same projected vis tokens as Trunk 1, just placed at a different sequence position.

Pseudocode for Variant C 4-trunk

class LLaVAStyle4TrunkVLM(nn.Module):
    def __init__(self, vit_config, llm):
        # Trunk 1: trainable vision encoder + 2-layer MLP projection (LLaVA-1.5 style)
        self.vit = ViT(**vit_config)
        self.vis_proj = nn.Sequential(
            nn.Linear(768, 4096), nn.GELU(), nn.Linear(4096, 4096),
        )
        # Frozen LLM (Trunks 2 + 4 collapse into the same frozen stack)
        self.llm = llm
        for p in self.llm.parameters():
            p.requires_grad = False
        # Trunk 3: NO new module (mechanism α)

    def forward(self, image, prompt_tokens, answer_tokens):
        # Trunks 1 + 3 share the same encoder pass under mechanism α
        v = self.vis_proj(self.vit(image))             # [B, 196, 4096]
        p = self.llm.embed(prompt_tokens)              # [B, N_prompt, 4096]
        a = self.llm.embed(answer_tokens)              # [B, N_ans,    4096]

        # Concat in causal trunk order: [v1 | p | v2 | a],  v2 := v
        seq = torch.cat([v, p, v, a], dim=1)
        N1, Np, N2, Na = 196, p.shape[1], 196, a.shape[1]
        mask = build_4trunk_mask(N1, Np, N2, Na)
        h = self.llm.layers(seq, attn_mask=mask)
        logits = self.llm.lm_head(h)
        # Loss only on the answer slice
        return logits[:, N1 + Np + N2 - 1 : -1, :]

def build_4trunk_mask(N1, Np, N2, Na):
    """Block-causal mask with bidirectional attention inside non-answer trunks."""
    L = N1 + Np + N2 + Na
    m = torch.zeros(L, L, dtype=torch.bool)
    # Trunk boundaries
    s1, s2, s3, s4 = 0, N1, N1 + Np, N1 + Np + N2
    e1, e2, e3, e4 = s2, s3, s4, L
    # Bidirectional within each non-answer trunk
    m[s1:e1, s1:e1] = True
    m[s2:e2, s2:e2] = True
    m[s3:e3, s3:e3] = True
    # Causal between trunks (lower-triangular at trunk granularity)
    m[s2:e2, s1:e1] = True            # prompt    sees vis_1
    m[s3:e3, s1:e2] = True            # vis_2     sees vis_1 + prompt
    m[s4:e4, s1:e3] = True            # answer    sees all earlier trunks
    # Causal within the answer trunk
    m[s4:e4, s4:e4] = torch.tril(torch.ones(Na, Na, dtype=torch.bool))
    return m

Trainable params: vision encoder (~86M) + vis_proj (~17M) ≈ ~103M — the same as 3-trunk Variant C. Trunk 3 adds zero parameters because it reuses the Trunk 1 encoding.

Why this works at all. Even though vis_tokens_2 equals vis_tokens_1 at the LLM input, the two diverge after layer 1: vis_2's queries can attend to prompt keys/values that vis_1's queries cannot. By the top of the LLM, vis_2 has been refined into a question-aware representation entirely by the LLM's own attention. This is the cheapest possible 4-trunk realization — no new modules, no extra encoder pass. If you want a richer Trunk 3, swap to Mechanism β (paying ~40M extra params for a real refiner stack); the rest of the LLaVA-style assembly is unchanged.

4. Variant A 4-trunk — Flamingo-style with three cross-attentions per layer

Trunk 2 is the lower 18 frozen LLM layers, producing prompt_features. Trunk 3 uses Mechanism β: a vision refiner with cross-attention to prompt_features produces a genuinely different vis_tokens_2. Trunk 4 is the upper 6 frozen LLM layers, augmented with three gated cross-attention modules per layer — one each for vis_tokens_1, prompt_features, and vis_tokens_2. All three gates initialize at zero so the model is identical to a frozen LLM at step 0.

Figure 4: Variant A 4-trunk. The decoder layer picks up a third gated cross-attention to vis_tokens_2 on top of the two from the 3-trunk version. All three gates start at zero, so the model boots up indistinguishable from the frozen LLM and gradually opens its eyes to (vis_1, prompt, vis_2) over the course of training.

Pseudocode for Variant A 4-trunk

class FlamingoStyle4TrunkVLM(nn.Module):
    def __init__(self, vit_config, llm, n_prompt=18, n_decoder=6, L_refiner=4):
        # Trunk 1
        self.vit = ViT(**vit_config)
        self.vis_proj = nn.Linear(768, 4096)

        # Trunk 2: lower 18 LLM layers, frozen
        self.embed = llm.embed; self.embed.requires_grad_(False)
        self.prompt_layers = llm.layers[:n_prompt]
        for p in self.prompt_layers.parameters():
            p.requires_grad = False

        # Trunk 3: vision refiner (Mechanism β)
        self.refiner = VisionRefiner(L_refiner=L_refiner, d=4096)

        # Trunk 4: 3 cross-attn modules + frozen LLM layer per decoder block
        self.decoder_blocks = nn.ModuleList()
        for i in range(n_decoder):
            llm_layer = llm.layers[n_prompt + i]
            for p in llm_layer.parameters():
                p.requires_grad = False
            self.decoder_blocks.append(nn.ModuleDict({
                "x_vis1":   GatedCrossAttention(d=4096, kv_d=4096),  # NEW
                "x_prompt": GatedCrossAttention(d=4096, kv_d=4096),  # NEW
                "x_vis2":   GatedCrossAttention(d=4096, kv_d=4096),  # NEW
                "llm_layer": llm_layer,                              # FROZEN
            }))
        self.lm_head = llm.lm_head; self.lm_head.requires_grad_(False)

    def encode_vision_1(self, image):
        return self.vis_proj(self.vit(image))                # [B, 196, 4096]

    def encode_prompt(self, prompt_tokens):
        h = self.embed(prompt_tokens)
        for layer in self.prompt_layers:
            h = layer(h, attn_mask="causal")
        return h                                              # [B, N_prompt, 4096]

    def encode_vision_2(self, vis_tokens_1, prompt_features):
        return self.refiner(vis_tokens_1, prompt_features)    # [B, 196, 4096]

    def decode_step(self, vis_1, prompt_features, vis_2, answer_so_far):
        h = self.embed(answer_so_far)                         # [B, K, 4096]
        for block in self.decoder_blocks:
            h = block["x_vis1"](h, vis_1)
            h = block["x_prompt"](h, prompt_features)
            h = block["x_vis2"](h, vis_2)
            h = block["llm_layer"](h, attn_mask="causal")
        return self.lm_head(h[:, -1, :])

Trainable params: vision encoder (~86M) + vis_proj (~3M) + refiner (~40M) + 6 × (3 × cross-attention ≈ 33M each) ≈ ~725M. The largest of the three variants — the price of three independent attention paths into the decoder.

5. Variant B 4-trunk — Cambrian per-block with two Q-Formers per layer

Trunk 2 is the lower 18 frozen LLM layers (same as A). Trunk 3 uses Mechanism β with the same vision refiner. Trunk 4: at every decoder layer, instantiate two Q-Formers — one for vis_tokens_1, one for vis_tokens_2 — each producing M=64 compressed visual tokens. Concatenate everything into the LLM sequence at that layer: [vis_compressed_1 | prompt_features | vis_compressed_2 | answer].

Figure 5: Variant B 4-trunk. Each decoder layer now hosts two independent Q-Formers — one looking at the problem-blind vis_tokens_1, one at the prompt-aware vis_tokens_2. Their outputs are placed on either side of the prompt features in the LLM sequence, preserving the causal trunk ordering (v₁ ≺ p ≺ v₂ ≺ answer).

Pseudocode for Variant B 4-trunk

class CambrianPerBlock4TrunkVLM(nn.Module):
    def __init__(self, vit_config, llm, M=64, n_prompt=18, n_decoder=6, L_refiner=4):
        # Trunk 1
        self.vit = ViT(**vit_config)
        self.vis_proj = nn.Linear(768, 4096)
        # Trunk 2: lower 18 LLM layers, frozen
        self.embed = llm.embed; self.embed.requires_grad_(False)
        self.prompt_layers = llm.layers[:n_prompt]
        for p in self.prompt_layers.parameters():
            p.requires_grad = False
        # Trunk 3: vision refiner
        self.refiner = VisionRefiner(L_refiner=L_refiner, d=4096)
        # Trunk 4: two Q-Formers + frozen LLM layer per block
        self.decoder_blocks = nn.ModuleList()
        for i in range(n_decoder):
            llm_layer = llm.layers[n_prompt + i]
            for p in llm_layer.parameters():
                p.requires_grad = False
            self.decoder_blocks.append(nn.ModuleDict({
                "qformer1":  QFormer(M=M, d=4096, kv_d=4096),  # NEW (vis_1)
                "qformer2":  QFormer(M=M, d=4096, kv_d=4096),  # NEW (vis_2)
                "llm_layer": llm_layer,                         # FROZEN
            }))
        self.M = M
        self.lm_head = llm.lm_head; self.lm_head.requires_grad_(False)

    def encode_vision_1(self, image):
        return self.vis_proj(self.vit(image))

    def encode_prompt(self, prompt_tokens):
        h = self.embed(prompt_tokens)
        for layer in self.prompt_layers:
            h = layer(h, attn_mask="causal")
        return h

    def encode_vision_2(self, vis_1, prompt_features):
        return self.refiner(vis_1, prompt_features)

    def decode_step(self, vis_1, prompt_features, vis_2, answer_so_far):
        ans = self.embed(answer_so_far)
        Np = prompt_features.shape[1]
        for block in self.decoder_blocks:
            v1c = block["qformer1"](vis_1)               # [B, M, 4096]
            v2c = block["qformer2"](vis_2)               # [B, M, 4096]
            seq = torch.cat([v1c, prompt_features, v2c, ans], dim=1)
            mask = build_4trunk_mask(self.M, Np, self.M, ans.shape[1])
            seq = block["llm_layer"](seq, attn_mask=mask)
            ans = seq[:, self.M + Np + self.M:, :]      # carry only answer slice forward
        return self.lm_head(ans[:, -1, :])

Trainable params: vision encoder (~86M) + vis_proj (~3M) + refiner (~40M) + 6 × 2 Q-Formers (~50M each) ≈ ~730M. Comparable in size to Variant A; slightly larger because of the doubled Q-Former count.

Which to start with?

  • Run Variant C 4-trunk first, against Variant C 3-trunk as the control. The question this directly tests: does putting the image into the LLM sequence twice (with prompt in the middle) help, even with no new parameters? If yes, it's a striking and cheap result. If not, you've ruled out the cheapest version of the 4-trunk hypothesis before paying for refiners or per-block injection.
  • Then test Mechanism β alone in Variant C (LLaVA-style, with a real refiner). This isolates "is a prompt-aware refiner useful?" from "are per-block cross-attn / Q-Former bridges useful?"
  • Promote to Variant A or B only if the C-with-refiner result is strong but you suspect the 24-layer-frozen-LLM bottleneck is now what's holding things back.

7. Open questions specific to 4-trunk

  1. Does Mechanism α (free) actually do anything useful? The whole 4-trunk-Variant-C bet rests on the LLM being able to refine vis_2 via its own attention. An ablation: train Variant C 3-trunk and Variant C 4-trunk-α on identical data; if the 4-trunk version wins, the bet pays off.
  2. How deep should the refiner be? L_refiner = 4 is a guess. Sweep {0, 1, 2, 4, 8} — 0 reduces to mechanism α, 8 starts to look like a small encoder of its own.
  3. Should Trunk 3 share weights with Trunk 1? A second-pass-with-shared-encoder would actually re-run the ViT (Falcon-Perception-style) instead of building a separate refiner. More expensive per step but could be more parameter-efficient overall.
  4. Cross-attention asymmetry in the refiner. In Variant C the refiner cross-attends to prompt_embeds (raw token embeddings, no LLM processing). In A/B it cross-attends to prompt_features (after 18 LLM layers). Does the encoded version actually help, or are raw embeddings enough?
  5. Sequence-length blowup in Variant C. 196 + 6 + 196 + K is ~400 tokens for the LLM at every layer. For long contexts or multi-image inputs this scales poorly. Worth measuring whether mechanism α helps enough to justify the doubled sequence cost.
  6. Causal-vs-bidirectional inside Trunk 3. Should vis_2 attend bidirectionally over its own 196 tokens, or causally? Bidirectional is the default and matches the Trunk 1 convention.
  7. Multi-pass extension (5-, 6-, … N-trunk). If Trunk 3 helps, does Trunk 5 (image-after-partial-answer) help further? See section 8.

8. The path to N-trunk (preview)

The 4-trunk recipe extends to longer chains by interleaving more vision-refiner passes between text trunks. A natural 5-trunk arrangement: [Image | Prompt | Image' | partial-Answer | Image''], where each successive Image-trunk is a refiner pass that takes the previous vision tokens and the most recent text trunk as cross-attention K/V. This starts to resemble a chain-of-thought over visual primitives (see DeepSeek's "Thinking with Visual Primitives" and the Visual CoT survey) — each visual trunk corresponds to one "look back at the image" step in the reasoning chain.

For each variant the extension is mechanical:

  • Variant C: add another bidirectional block to the LLM input sequence; the mask just adds another row/column to the trunk-causal block matrix. Free if Mechanism α; pay another refiner stack if Mechanism β.
  • Variant A: add another gated cross-attention per decoder layer for each new vision trunk. Linear growth in cross-attn count.
  • Variant B: add another Q-Former per decoder layer for each new vision trunk. Linear growth in Q-Former count.

Beyond ~5 trunks, the per-decoder-layer module count in A and B becomes onerous; Variant C scales most gracefully (just a longer input sequence, no new modules), at the cost of a longer LLM context per step.
