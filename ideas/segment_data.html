VLM Pretraining Notes

架构：PaliGemma-style  |  关注任务：Detection / Grounded Captioning（Seg 暂缓）

1. Pretraining 任务格式

Detection / Referring Expression（ARef）

prompt:  detect cat ; dog
label:   <loc0382><loc0201><loc0695><loc0478> cat <loc0156><loc0089><loc0401><loc0312> dog

每个 object 输出 4 个 <loc> token（顺序：ymin xmin ymax xmax）+ class name。多个 object 按 y_min 从小到大排序（top-to-bottom）。

Grounded Captioning（GCap，来自 LocCa）

prompt:  caption <loc0382><loc0201><loc0695><loc0478>
label:   a cat sitting on a wooden table

给定 bbox 作为 prompt 的一部分，输出框内区域的自然语言描述。需要 region-level caption 数据，不能用整图 caption。

Segmentation（暂缓）

label:   <loc>x4  <seg000>...<seg127> x16  cat

16 个 <seg> token 是 VQ-VAE（codebook size=128）对 bbox 内 64×64 binary mask 的编码。VQ-VAE 权重开源于 google-research/big_vision，但集成链路复杂。先跑 box-only，之后再加。

2. <loc> Token 机制

向词表新增 1024 个 token（<loc0000>~<loc1023>），表示归一化坐标 bin 后的整数值（假设图片 resize 到 1024×1024）。与实际输入分辨率无关。Embedding 随训练学习，初始化建议用 AvgEmb（已有词表均值 + 小噪声）。

def coord_to_loc(coord_normalized):   # coord in [0.0, 1.0]
    return int(coord_normalized * 1023)

# Visual Genome box format: x,y = top-left, w,h = size
def vg_box_to_loc_tokens(x, y, w, h, img_w, img_h):
    ymin = int((y        / img_h) * 1023)
    xmin = int((x        / img_w) * 1023)
    ymax = int(((y+h)    / img_h) * 1023)
    xmax = int(((x+w)    / img_w) * 1023)
    return f"<loc{ymin:04d}><loc{xmin:04d}><loc{ymax:04d}><loc{xmax:04d}>"

3. LocCa 数据来源（原版 vs 开源复现）

原版用 Google 内部 WebLI（1B pairs，不开源）+ OWL-ViT 生成伪 box 标注（confidence > 0.3 过滤）。

开源复现建议： CC3M / CC12M（有 alt-text）+ Grounding DINO（开源）→ 对 alt-text 里的名词短语生成 pseudo box。可将 GCap 数据量从 Visual Genome 的 108K 图扩展到 3M~12M 量级，处理 CC3M 全量约需几十 GPU hours。

4. 开源数据集

数据集 规模 Detection Grounded Caption 备注
Visual Genome 108K 图 / 3.8M regions ✓ objects.json ✓ region_descriptions.json GCap 最直接可用；caption 较短（均~5词）
Objects365 2M 图 / 30M boxes / 365类 ✓ 最推荐 ✗ 无 caption 类别多、质量高；LocCa 论文用它预训练 detector
OpenImages v7 9M 图 / 16M boxes / 600类 ✓ ✗ 无 caption 规模最大；含 visual relationship 标注
SA-1B (SAM) 11M 图 / 1B masks △ 需补 label ✗ 无 text label；用于 seg task（当前跳过）
COCO 118K 图 / 860K boxes / 80类 ✓ △ 整图 caption，非 region 小但质量高；主要用于 eval

5. Visual Genome 详细说明

下载地址：homes.cs.washington.edu/~ranjay/visualgenome/data/dataset

与你之前用的 VQA 是同一份图片，不同标注文件：

  • region_descriptions.json.zip — Grounded captioning 用
  • objects.json.zip — Detection 用
  • question_answers.json.zip — 你之前用的 VQA

region_descriptions.json 结构

[
  {
    "id": 1,
    "regions": [
      {
        "region_id": 1382,
        "image_id": 1,
        "phrase": "the man is wearing a blue shirt",
        "x": 251,   // top-left x
        "y": 10,    // top-left y
        "width": 108,
        "height": 269
      }, ...
    ]
  }, ...
]
预处理注意：
  • 部分 box 坐标超出图片边界 → clip 到 [0, img_w/h]
  • 部分 phrase 为空或单个词 → 过滤 len(phrase.split()) < 2
  • box 格式是 x,y,w,h（左上角+宽高），不是 xyxy
  • PaliGemma 坐标顺序是 ymin xmin ymax xmax（先 y 后 x）

完整预处理函数

def process_vg_region(region, img_w, img_h):
    x, y = region['x'], region['y']
    w, h = region['width'], region['height']
    phrase = region['phrase'].strip()

    if len(phrase.split()) < 2:
        return None
    if w <= 0 or h <= 0:
        return None

    # Clip to image bounds
    x  = max(0, min(x,     img_w))
    y  = max(0, min(y,     img_h))
    x2 = max(0, min(x + w, img_w))
    y2 = max(0, min(y + h, img_h))

    ymin = int((y  / img_h) * 1023)
    xmin = int((x  / img_w) * 1023)
    ymax = int((y2 / img_h) * 1023)
    xmax = int((x2 / img_w) * 1023)

    loc = f"<loc{ymin:04d}><loc{xmin:04d}><loc{ymax:04d}><loc{xmax:04d}>"

    return {
        'gcap_prompt': f"caption {loc}",
        'gcap_label':  phrase,
        'aref_prompt': phrase,
        'aref_label':  loc,
    }
