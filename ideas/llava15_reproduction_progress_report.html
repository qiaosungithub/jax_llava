Remaining Differences From LLaVA-1.5

AxisCurrent status
LM size and initializationOfficial LLaVA-1.5 7B target uses Vicuna-v1.5-7B. Our first/current reproduction uses Gemma3-1B-IT, so the language model is much smaller and instruction-tuned under a different tokenizer/chat distribution.
Vision towerBoth use CLIP-L/14 at 336px in spirit. Our implementation uses OpenAI CLIP-L/14-336 with feature layer -2, patch features, MLP2x GELU projector, and JAX/HSDP. Image preprocessing is not an exact official-code reproduction because we use stretch resize.
Stage-1 dataOfficial target is LLaVA-Pretrain/LCS-558K. First run used blip3o-short; current config uses cc12m. Both are replacement data, not exact data-level reproduction.
Stage-2 sample definitionOfficial LLaVA-1.5 reports a 665K SFT recipe. Our training also uses 5195 x 128 = 664,960 stage-2 samples, but samples from a much larger LLaVA-OV1.5 pool plus weighted academic datasets. It is recipe-scale-aligned, not exact sample-list-aligned.
First-run missing SFT categoriesFirst run lacked OKVQA, A-OKVQA, OCRVQA, Visual Genome detection/grounded boxes, RefCOCO grounding, and ShareGPT text-only.
Current config remaining data gapsCurrent config adds OKVQA/A-OKVQA/OCRVQA/VG detection/RefCOCO, but still does not use exact LLaVA-1.5 665K JSON/conversations and still omits ShareGPT text-only.
Extra/different dataLLaVA-OV1.5-Instruct is newer and much larger than LLaVA-1.5 665K. This may improve breadth but makes the result an ablation-style reproduction rather than a strict LLaVA-1.5 data reproduction.
Eval protocolWe track several official metrics. POPE is aligned to the adversarial split in the spreadsheet: final logs contain both macro F1 and adversarial F1, and row 148 records the adversarial F1. We do not yet track MMBench-CN, SEED all/video, LLaVA-Wild, or MM-Vet in row 148. We additionally track VStar/OCRBench/MMVP/CountBench/RefCOCOg/KNN, which are not original LLaVA-1.5 paper-table metrics.
