VLM Baseline Evaluation 数据去重检查计划

目标：确认本机训练/混合数据是否与以下 evaluation sets 发生泄漏： TextVQA val, V*, MMVP, OCRBench, CountBenchQA, VQAv2 val, MME, POPE, RefCOCOg。
判定口径：不仅检查同名 dataset subset，还要检查 source image pool overlap、question/answer overlap、OCR text overlap、以及 COCO-family image id overlap。

0. 需要先明确的目录结构

建议在服务器上新建一个统一工作目录，例如：

/path/to/dedup_work/
  manifests/
  hashes/
  reports/
  scripts/
  logs/

准备两个输入集合：

  1. Train-side datasets：你们 VLM baseline 实际训练/finetune 用到的所有 dataset，包括 LLaVA-1.5-OV 相关子集、LLaVA-1.5 instruct、FineVision、OCR/VQA/COCO/GQA/AOKVQA/TallyQA 等。
  2. Eval-side datasets：上述九个 benchmark 的 evaluation split，尤其是 val 或官方 test/dev split。

最终每个 sample 都统一转成 manifest JSONL：

{
  "dataset": "textvqa_val",
  "split": "val",
  "sample_id": "...",
  "image_path": "/abs/path/to/image.jpg",
  "image_id": "...",
  "source_image_id": "COCO_train2014_000000123456",
  "question": "...",
  "answer": "...",
  "answers": ["..."],
  "metadata": {...}
}

1. 去重层级与判定标准

层级 检查内容 判定 优先级
Exact image bytes 文件 md5/sha256 完全一致 确定 image leakage 最高
Perceptual image hash pHash/dHash/aHash/Hamming distance 很小 高度疑似同图、resize、re-encode、crop 最高
Dataset image id COCO image_id、TextVQA image_id、GQA image_id 等相同 确定 source image overlap 最高
Question text normalized question 完全一致或近似一致 question leakage；若图像也相同则更严重 高
Answer text question+answer pair 完全一致 annotation leakage 高
Source dataset membership 训练集包含同名或同源 benchmark dataset 需要进一步 split-level / image-level 过滤 中高
推荐最终报告口径：
  • strict contaminated：eval image 与 train image 在 exact hash、perceptual hash、或 canonical image_id 上命中。
  • annotation contaminated：eval question/answer pair 在 train-side instruction data 中命中。
  • domain/source overlap only：同源数据集存在，但未发现 eval image/question 级别命中。
  • clean under tested criteria：上述检查均未命中。

2. Manifest 构建方案

2.1 统一解析所有训练数据

对每个训练 dataset 输出一个 JSONL manifest。关键字段必须包括：

  • dataset, split, sample_id
  • image_path 或 image_bytes_key
  • image_id：原始数据集 id，例如 COCO image id、TextVQA image id、GQA image id
  • source_dataset：COCO / TextVQA / GQA / AOKVQA / OCRVQA / TallyQA 等
  • question, answer, conversation

2.2 统一解析 evaluation sets

每个 eval set 单独输出 manifest：

manifests/eval_textvqa_val.jsonl
manifests/eval_vstar.jsonl
manifests/eval_mmvp.jsonl
manifests/eval_ocrbench.jsonl
manifests/eval_countbenchqa.jsonl
manifests/eval_vqav2_val.jsonl
manifests/eval_mme.jsonl
manifests/eval_pope.jsonl
manifests/eval_refcocog.jsonl

3. Hash 计算方案

3.1 Exact hash

对所有 train/eval images 计算：

  • sha256_file：原始文件 bytes hash
  • md5_file：方便与一些 dataset metadata 对齐

3.2 Canonical image hash

避免同图不同编码导致漏检。统一读图后转 RGB，再 resize 到固定大小，例如 512x512 或 256x256，然后计算 hash：

  • sha256_rgb_512：RGB canonical bytes hash
  • phash, dhash：感知哈希，捕捉 resize / jpeg re-encode / minor crop
# scripts/compute_hashes.py 的核心逻辑
from PIL import Image
import hashlib
import imagehash

img = Image.open(image_path).convert("RGB")
img512 = img.resize((512, 512), Image.BICUBIC)
sha256_rgb_512 = hashlib.sha256(img512.tobytes()).hexdigest()
phash = str(imagehash.phash(img, hash_size=16))
dhash = str(imagehash.dhash(img, hash_size=16))

输出：

hashes/train_hashes.jsonl
hashes/eval_hashes.jsonl

3.3 Perceptual hash 检索

先 exact match，再 pHash 近邻搜索。对 16x16 hash，Hamming distance 推荐阈值：

  • <= 4：几乎确定同图或极轻微处理
  • 5-8：高度疑似，需要人工抽查
  • 9-12：可能是近似图、crop 或同场景，建议只作为 secondary evidence

4. Text / QA 去重方案

4.1 Normalize 函数

对 question、answer、conversation 统一 normalize：

  • lowercase
  • 去标点
  • 合并空格
  • 英文数字规范化，例如 two → 2，可选
  • 去除 VQA prompt 模板，例如 answer en ...
def normalize_text(s):
    s = s.lower().strip()
    s = re.sub(r"[^a-z0-9\s]", " ", s)
    s = re.sub(r"\s+", " ", s)
    return s

4.2 检查项

检查key意义
Question exact match norm_question 检测 eval question 是否被 train-side instruction 数据见过
Question + answer exact match norm_question + "\t" + norm_answer 检测 annotation-level leakage
Image + question match image_id/hash + norm_question 最强的 sample-level leakage evidence
Fuzzy question match MinHash / simhash / rapidfuzz 检测 paraphrase 或模板化问题

6. 推荐脚本结构

scripts/
  build_manifest_train.py        # 解析所有训练数据，输出 train manifest
  build_manifest_eval.py         # 解析九个 eval benchmark，输出 eval manifests
  compute_image_hashes.py        # exact hash + canonical hash + pHash/dHash
  dedup_by_ids.py                # COCO/TextVQA/GQA/AOKVQA/source id 对齐
  dedup_by_exact_hash.py         # sha256/md5 exact match
  dedup_by_phash.py              # pHash Hamming search
  dedup_by_text.py               # question/answer exact + fuzzy match
  summarize_report.py            # 汇总成 csv/html/json
  sample_for_manual_review.py    # 抽取疑似重复图像 pair 生成人工检查网页

7. 具体执行顺序

  1. 列出训练数据清单：所有参与 baseline training/finetuning 的 JSON、JSONL、WebDataset tar、image folder、HF cache。
  2. 构建 train manifest：每条训练样本都能追踪到 image path/source id/question/answer/conversation。
  3. 构建 eval manifest：九个 benchmark 每条样本都保留 canonical id。
  4. 先做 ID-level check：COCO/TextVQA/GQA/AOKVQA/OCRVQA/TallyQA 等 id 直接查交集。
  5. 再做 exact hash check：sha256/md5/canonical RGB hash。
  6. 再做 pHash check：处理 resize、JPEG 重编码、轻微 crop。
  7. 再做 text-level check：question、answer、question+answer、image+question。
  8. 人工抽查：对 pHash 距离 5-12 的样本生成 side-by-side HTML。
  9. 生成最终报告：每个 eval set 给出 image-level / annotation-level / source-level overlap 数量和比例。

9. 建议的最终 claim 模板

如果完全没有 image-level / annotation-level 命中

We checked all evaluation images and annotations against the training mixture using exact file hashes, canonical RGB hashes, perceptual hashes, source dataset IDs, and normalized question-answer matching. Under these criteria, we found no image-level or annotation-level overlap for [dataset list]. Some benchmarks share broad source domains with common VLM training data, but no evaluated sample was matched to the training set.

如果存在 source-image overlap，但没有 QA overlap

We found source-image overlap for [dataset], mainly because both the training mixture and the benchmark draw from [COCO/TextVQA/etc.]. We therefore report this benchmark separately and avoid claiming it as strictly unseen. No exact question-answer pair overlap was found after text normalization.

如果发现 eval image 或 QA pair 泄漏

We found [N] evaluation samples with image-level or annotation-level overlap with the training mixture. These samples were removed from evaluation / the metric was recomputed on the de-duplicated subset / the benchmark is marked as contaminated and excluded from the main comparison.

10. 最小可行版本

如果时间有限，优先做以下四步：

  1. 构建 train/eval manifest，保留 image path、source id、question、answer。
  2. 对 TextVQA val、VQAv2 val、POPE、RefCOCOg 做 source image id 检查，特别是 COCO image_id。
  3. 对所有 eval images 做 exact hash + pHash 检查。
  4. 对所有 eval question/answer 做 normalized exact match。

注：这份计划的核心原则是“不要只看 subset 名字”。VLM 数据污染更常见的是 source image pool overlap，例如 COCO images 被 VQAv2、RefCOCOg、POPE、AOKVQA、captioning/instruction 数据多次复用；OCR benchmark 也经常从 TextVQA/OCRVQA/ST-VQA/DocVQA/TextOCR 等源数据派生。
