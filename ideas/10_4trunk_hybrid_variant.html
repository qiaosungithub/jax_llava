4-Trunk VLM — Hybrid Design (Image · Prompt · Image-again · Answer)

Extending the 3-trunk hybrid from brainstorm/07 with a second, prompt-aware vision encoding

Context for this document. Brainstorm/07 proposed a 3-trunk hybrid: ViT + Q-Former + Linear → frozen lower-16 LLM blocks (text encoder for [prompt | answer]) → frozen upper-8 LLM blocks (decoder). This document extends that recipe to 4 trunks: the image is "looked at twice" — first problem-blind (Trunk 1, identical to brainstorm/07), then again after the prompt has been read (Trunk 3, the new piece). The 4 trunks, in causal order, are:
  • Trunk 1 — Image (problem-blind): ViT → Q-Former₁ → Linear → img_features_1. Same as brainstorm/07.
  • Trunk 2 — Text encoder: lower 16 frozen LLM blocks on [prompt | answer] → text_features. Same as brainstorm/07.
  • Trunk 3 — Image (prompt-aware) — NEW: a second Q-Former with cross-attention to prompt_features takes the ViT output and produces a refined img_features_2.
  • Trunk 4 — Decoder: upper 8 frozen LLM blocks on concat(img_features_1, prompt_features, img_features_2, answer_features). Same backbone as brainstorm/07; just a longer concatenated input.
The trunks are causally ordered: each can attend to all earlier trunks but not to future ones. Compared to brainstorm/07, the only new trained module is the second Q-Former with prompt cross-attention; the frozen-LLM property and the depth-split structure are preserved.
Running example used throughout (same as brainstorm/07):
  • Image: 3 × 224 × 224 RGB photo of a black cat
  • Prompt: "What color is the cat?" → L_txt = 6 tokens
  • Target answer: "The cat is black." → L_ans = 5 tokens

Intuition for the 4th trunk. Trunk 1 produces a problem-blind compressed view of the cat-on-chair scene. After the prompt is encoded in Trunk 2, Trunk 3 re-runs vision compression with prompt awareness — its 64 query tokens now know "the question is about color," so they can produce visual features more focused on cat-color-relevant regions. Trunk 4 then generates the answer using both views (broad first look + focused second look) plus the prompt.

1. Pipeline at a glance

(Trunk 1) Image (problem-blind)
     (3, 224, 224)
  → ViT-Large + Q-Former₁ + Linear  (TRAINED)
                    →  img_features_1   (L_img = 64,  D_txt = 4096)

(Trunk 2) Text encoder
     [prompt | answer]               # 6 + 5 = 11 ids
  → embed → LLM blocks 1 → 16 (FROZEN, causal)
                    →  text_features   (11, 4096)
        split:  prompt_features = text_features[:6]
                answer_features = text_features[6:]

(Trunk 3) Image (prompt-aware)  ← NEW
     reuse ViT output  +  prompt_features
  → Q-Former₂  (TRAINED, 4 blocks; queries cross-attend to ViT output and to prompt_features)
  → Linear₂  (TRAINED)
                    →  img_features_2   (L_img = 64,  D_txt = 4096)

(Trunk 4) Decoder
     concat(img_features_1, prompt_features, img_features_2, answer_features)
                    →  (64 + 6 + 64 + 5, 4096)  =  (139, 4096)
  → LLM blocks 17 → 24 (FROZEN, prefix-LM mask with trunk-causal ordering)
  → LM Head (FROZEN)
                    →  next-token logits on the answer slice (NTP loss)

Three things to flag up front:

  • Only Trunk 3 is new. Trunks 1, 2, 4 are structurally identical to brainstorm/07; just the decoder's input sequence is longer because img_features_2 sits between the prompt and the answer.
  • Q-Former₂ is the only new trained module. It re-uses the ViT output from Trunk 1 (no second ViT pass), and it cross-attends to prompt_features (no second text encoder pass). So the only extra training cost is ~30M params for the 4-block Q-Former₂.
  • The frozen-LLM property and the depth-split are preserved. The full LLM (embed, all 24 blocks, LM head) stays frozen. End-to-end the text path traverses all 24 LLM blocks; the image path traverses only the upper 8.

2. Trunks 1 and 2 — recap (unchanged from brainstorm/07)

For full detail of Trunks 1 and 2 see brainstorm/07. Here's just the summary of what they produce:

Recap of brainstorm/07 outputs: img_features_1 (64, 4096) from Trunk 1, and text_features (11, 4096) = [prompt_features | answer_features] from Trunk 2. The intermediate ViT output (256, 1024) is also re-used by Trunk 3 below — no second ViT pass.

3. Trunk 3 — Prompt-aware vision re-encoding (NEW)

Trunk 3's job is to produce a second compressed visual sequence that has been refined with knowledge of the prompt. Architecturally, it is a Q-Former with one extra cross-attention per block — the queries now attend to both the ViT output (the visual content) and prompt_features (the question context). Output shape matches img_features_1 so the decoder treats both visual sequences uniformly.

Figure 1: Trunk 3, top-to-bottom. Two input sequences arrive: the intermediate ViT output (256, 1024) from Trunk 1 and prompt_features (6, 4096) from Trunk 2. The new Q-Former₂ has its own 64 learnable queries and runs 4 transformer-style blocks; each block does self-attention over the queries, cross-attention from queries to ViT output (visual content), cross-attention from queries to prompt_features (question context), and an FFN. The output is projected via a separate Linear₂ to img_features_2 (64, 4096) — same shape as img_features_1 from Trunk 1, but now refined with knowledge of the question. Trainable params added: ~30M Q-Former₂ + ~4M Linear₂.

3.1 Cheaper alternative: Mechanism α (no extra params)

If you want the cheapest possible 4-trunk extension, you can set img_features_2 = img_features_1 and skip Q-Former₂ entirely (zero new parameters; Trunk 3 just re-references Trunk 1's output). The two visual encodings then start identical at the decoder input but diverge inside the upper 8 LLM blocks because they sit at different sequence positions: img_features_1 attends to nothing else, while img_features_2 attends to prompt_features via the LLM's self-attention. This mirrors Mechanism α from brainstorm/06 and is the strict zero-cost baseline. The pseudocode and figures below show the full Mechanism β version (with Q-Former₂); switching to α just deletes Q-Former₂ and aliases img_features_2 to img_features_1.

4. Trunk 4 — Decoder with 4-way concat

Same upper 8 frozen LLM blocks as in brainstorm/07. The only change is that the input is now a four-way concatenation in causal trunk order: [img_features_1 | prompt_features | img_features_2 | answer_features]. The mask preserves the trunk ordering so each trunk only sees earlier ones (and within the answer, attention is causal).

Figure 2: Trunk 4 (decoder), top-to-bottom. Four input sequences arrive — the two image features (light green from Trunk 1, dark green from Trunk 3) and the prompt/answer slices (blue and orange) of text_features. They are concatenated in causal trunk order into a 139-token sequence at D=4096 and pushed through the upper 8 frozen LLM blocks. The mask preserves the trunk ordering: img_features_1 attends only to itself; prompt_features attends to img_1; img_features_2 attends to img_1 + prompt; answer_features is causal and attends to all three earlier trunks. Only the orange answer slice is fed to the LM head.

5. End-to-end pipeline

Figure 3: End-to-end 4-trunk pipeline. Top row: the same Trunk 1 (image lane) and Trunk 2 (text lane) as in brainstorm/07, producing img_features_1 (64, 4096) and text_features (11, 4096). Middle: the new Trunk 3 lane — Q-Former₂ + Linear₂ takes the intermediate ViT output (re-used from Trunk 1) plus prompt_features (from Trunk 2) and produces a prompt-aware img_features_2 (64, 4096). Bottom: all four sequences are concatenated in causal trunk order into a 139-token sequence, run through the upper 8 frozen LLM blocks, and fed to the LM head. Trainable additions vs. brainstorm/07: ~30M for Q-Former₂ + ~4M for Linear₂.

6. Pseudocode

class Hybrid4TrunkVLM(nn.Module):
    def __init__(self, vit_config, llm,
                 P=14, L_low=16, L_img=64, D_img=1024, L_qf2=4):
        # ---- Trunk 1 (same as Hybrid3TrunkVLM from brainstorm/07) ----
        self.patch_embed = nn.Linear(3 * P * P, D_img)
        self.vit         = ViT(**vit_config)
        self.qformer1    = QFormer(M=L_img, d=D_img, kv_d=D_img)     # vision-only
        self.vis_proj1   = nn.Linear(D_img, llm.d_hidden)             # 1024 → 4096

        # ---- Trunk 2 (same: lower L_low frozen LLM blocks for [prompt|answer]) ----
        self.embed       = llm.embed
        self.text_layers = llm.layers[:L_low]
        for p in self.embed.parameters():       p.requires_grad = False
        for p in self.text_layers.parameters(): p.requires_grad = False

        # ---- Trunk 3 (NEW): Q-Former with prompt cross-attention ----
        self.qformer2    = QFormerWithPromptXAttn(
            M=L_img, d=D_img, kv_d_vis=D_img, kv_d_txt=llm.d_hidden,
            n_blocks=L_qf2,
        )
        self.vis_proj2   = nn.Linear(D_img, llm.d_hidden)

        # ---- Trunk 4 (same: upper LLM blocks + LM head, frozen) ----
        self.decoder_layers = llm.layers[L_low:]
        for p in self.decoder_layers.parameters(): p.requires_grad = False
        self.lm_head = llm.lm_head
        for p in self.lm_head.parameters(): p.requires_grad = False
        self.P = P

    # -------- Trunk 1 forward (also exposes the intermediate ViT output for Trunk 3) --------
    def encode_image_problem_blind(self, image):
        B = image.shape[0]
        patches = image.unfold(2, self.P, self.P).unfold(3, self.P, self.P)
        patches = patches.permute(0, 2, 3, 1, 4, 5).contiguous()
        patches = patches.view(B, -1, 3 * self.P * self.P)        # [B, 256, 588]
        h_vit = self.vit(self.patch_embed(patches))               # [B, 256, 1024]  (KEEP for Trunk 3)
        v1 = self.qformer1(h_vit)                                  # [B, L_img, 1024]
        img_features_1 = self.vis_proj1(v1)                        # [B, L_img, 4096]
        return img_features_1, h_vit

    # -------- Trunk 2 forward (causal pass over [prompt | answer]) --------
    def encode_text(self, text_tokens):
        h = self.embed(text_tokens)                                # [B, L_txt+L_ans, 4096]
        for layer in self.text_layers:
            h = layer(h, attn_mask="causal")
        return h                                                   # text_features

    # -------- Trunk 3 forward (NEW): prompt-aware vision re-encoding --------
    def encode_image_prompt_aware(self, h_vit, prompt_features):
        v2 = self.qformer2(h_vit, prompt_features)                 # [B, L_img, 1024]
        img_features_2 = self.vis_proj2(v2)                        # [B, L_img, 4096]
        return img_features_2

    # -------- Trunk 4 forward (training: teacher-forced) --------
    def forward(self, image, prompt_tokens, answer_tokens):
        L_p, L_a = prompt_tokens.shape[1], answer_tokens.shape[1]

        img_1, h_vit = self.encode_image_problem_blind(image)      # Trunk 1
        text = self.encode_text(torch.cat([prompt_tokens, answer_tokens], dim=1))  # Trunk 2
        prompt_features = text[:, :L_p, :]
        answer_features = text[:, L_p:, :]
        img_2 = self.encode_image_prompt_aware(h_vit, prompt_features)             # Trunk 3

        seq = torch.cat([img_1, prompt_features, img_2, answer_features], dim=1)   # [B, 139, 4096]
        L_i1, L_i2 = img_1.shape[1], img_2.shape[1]
        mask = build_4trunk_mask(L_i1, L_p, L_i2, L_a)
        for layer in self.decoder_layers:
            seq = layer(seq, attn_mask=mask)                       # frozen blocks 17..24

        logits = self.lm_head(seq)                                 # [B, 139, vocab]
        # answer slice for NTP, shifted
        start = L_i1 + L_p + L_i2
        return logits[:, start - 1 : -1, :]


def build_4trunk_mask(L_i1, L_p, L_i2, L_a):
    """Each non-answer trunk bidirectional within itself; causal between trunks; answer fully causal."""
    L = L_i1 + L_p + L_i2 + L_a
    m = torch.zeros(L, L, dtype=torch.bool)
    s1, s2, s3, s4 = 0, L_i1, L_i1 + L_p, L_i1 + L_p + L_i2
    e1, e2, e3, e4 = s2, s3, s4, L
    # Bidirectional within each non-answer trunk
    m[s1:e1, s1:e1] = True
    m[s2:e2, s2:e2] = True
    m[s3:e3, s3:e3] = True
    # Causal between trunks
    m[s2:e2, s1:e1] = True            # prompt sees img_1
    m[s3:e3, s1:e2] = True            # img_2  sees img_1 + prompt
    m[s4:e4, s1:e3] = True            # answer sees img_1 + prompt + img_2
    # Causal within the answer trunk
    m[s4:e4, s4:e4] = torch.tril(torch.ones(L_a, L_a, dtype=torch.bool))
    return m


class QFormerWithPromptXAttn(nn.Module):
    """Q-Former with one extra cross-attention per block (queries → prompt_features)."""
    def __init__(self, M, d, kv_d_vis, kv_d_txt, n_blocks=4, n_heads=16):
        super().__init__()
        self.queries = nn.Parameter(torch.randn(M, d) * 0.02)
        self.blocks = nn.ModuleList([
            QFormerBlockWithPrompt(d, kv_d_vis, kv_d_txt, n_heads) for _ in range(n_blocks)
        ])

    def forward(self, vit_output, prompt_features):
        # vit_output: [B, 256, kv_d_vis], prompt_features: [B, L_p, kv_d_txt]
        B = vit_output.shape[0]
        Q = self.queries.unsqueeze(0).expand(B, -1, -1)
        for block in self.blocks:
            Q = block(Q, vit_output, prompt_features)
        return Q


class QFormerBlockWithPrompt(nn.Module):
    def __init__(self, d, kv_d_vis, kv_d_txt, n_heads):
        super().__init__()
        self.self_attn   = SelfAttention(d, n_heads)
        self.x_attn_vis  = CrossAttention(d, kv_d_vis, n_heads)
        self.x_attn_text = CrossAttention(d, kv_d_txt, n_heads)
        self.ffn = SwiGLU(d, 4 * d)
        self.n1, self.n2, self.n3, self.n4 = (nn.LayerNorm(d) for _ in range(4))

    def forward(self, Q, vit_output, prompt_features):
        Q = Q + self.self_attn(self.n1(Q))                              # self-attn over queries
        Q = Q + self.x_attn_vis (self.n2(Q), kv=vit_output)             # cross-attn → vision
        Q = Q + self.x_attn_text(self.n3(Q), kv=prompt_features)        # cross-attn → prompt   (NEW vs. Q-Former₁)
        Q = Q + self.ffn(self.n4(Q))
        return Q

Trainable parameter footprint:

  • Trunk 1: patch_embed + ViT-Large + Q-Former₁ + vis_proj₁ ≈ ~335M (same as brainstorm/07)
  • Trunk 3 (new): Q-Former₂ (4 blocks) + vis_proj₂ ≈ ~30M + ~4M = ~34M
  • Trunk 2 + Trunk 4: 0 (LLM is fully frozen, including embed and LM head)
  • Total: ~370M on top of a fully frozen 24-layer LLM (vs. ~335M for the 3-trunk hybrid)

How to run the comparison

The 4-trunk hybrid is a strict superset of the 3-trunk hybrid. The cleanest ablation:

  • 3-trunk baseline = brainstorm/07 as-is
  • 4-trunk Mechanism α = same Trunks 1, 2, 4, with img_features_2 = img_features_1 (zero new params; tests pure positional re-encoding)
  • 4-trunk Mechanism β = this document (Q-Former₂ with prompt cross-attention; the full-strength version)

The α vs. β gap measures how much an explicit prompt-aware refiner buys you over the LLM's own attention pattern. The 3-trunk vs. (α or β) gap measures whether two visual encodings — at any cost — actually help.

8. Open questions

  1. How many blocks in Q-Former₂? 4 is a starting guess. Sweep {2, 4, 8} — 2 may be too shallow to integrate prompt context, 8 starts to overlap with the upper-8 decoder's job.
  2. Share queries with Q-Former₁? Could either (a) tie the 64 queries between Q-Former₁ and Q-Former₂ (fewer params, same "vocabulary" of visual abstractions), or (b) keep them separate (more flexible, more params). Default: separate.
  3. Cross-attend to prompt_features or to raw prompt_embeds? Default: prompt_features (after the lower 16 LLM blocks) — richer, mid-LLM representation. Alternative: raw embeddings (cheaper to compute, no waiting for Trunk 2).
  4. Should Q-Former₂ also see the partially-generated answer at inference? If yes, Trunk 3 becomes a function of (image, prompt, answer-so-far) — a much richer signal but requires re-running Q-Former₂ at every decoding step. Default: no; Trunk 3 sees only image + full prompt and is computed once per sample.
  5. Decoder mask: should img_features_2 attend to img_features_1? The default trunk-causal mask says yes (Trunk 3 sees Trunk 1). An alternative is to have Trunk 3 attend only to itself + prompt (forcing it to be a "pure refinement" of the prompt-conditioned visual content). Worth ablating.
  6. Where does the gradient come from? Q-Former₂ only gets gradient through the upper 8 frozen LLM blocks → answer slice. The signal could be weaker than for Q-Former₁ (which is closer to the input position in the decoder sequence). Sample efficiency may differ.
  7. Path to N-trunk. If Trunk 3 helps, does Trunk 5 (image-after-partial-answer) help further? Each new vision trunk adds another Q-Former (with cross-attention to all earlier text trunks). Linear growth in trained params; quadratic growth in decoder sequence length.

The 4-trunk hybrid extends brainstorm/07 by adding a second visual encoding that is aware of the prompt. The only structural change is one new trained module — Q-Former₂ with cross-attention to prompt_features — and one extra projection. Trunks 1, 2, and 4 are unchanged; Trunk 4's input sequence just gets longer (75 → 139 tokens). The frozen-LLM property and the depth-split structure are preserved. Trainable budget grows from ~335M to ~370M. The clean ablation (3-trunk vs. 4-trunk-α vs. 4-trunk-β) measures whether a second, prompt-aware vision pass provides signal over and above what the LLM's own attention already extracts.
