Four-Trunk VLM: Prompt, Pixel Patches, Registers 的第二次看图设计

这份 report 重新整理 5 个 Trunk-3 架构，重点吸收两个新约束：第一次粗看特征和第二次 fine patch embedding 在 channel 维度融合；第二次最终仍输出长度受控的 1D register features。

核心结论。 这次我会把 Trunk 3 的问题抽象成三类 token 如何交互： image patches 提供高低层视觉证据， prompt tokens 提供“要找什么”的语义， register tokens 决定最终 1D feature 的长度和槽位。 最重要的设计点不是“有没有 cross-attention”，而是 prompt 如何先形成 word-to-patch 的定位信号，再让 registers 从这些被定位和上下文化后的 patches 中抽取 compact features。

1. 新的公共前处理: 粗看 + 细看 + Channel Concatenation

你提到的 channel concat 很关键，因为它把原来的三 stream 问题先压成一个更干净的 image stream。对于 512 x 512 图像，一个自然设定是：

  • 第一次看图: patch size 32，得到 16 x 16 = 256 个 coarse tokens。它们负责全局语义和对象级上下文。
  • 第二次看图: patch size 16，得到 32 x 32 = 1024 个 fine tokens。它们负责文字、小物体、局部纹理、颜色和空间细节。
  • 对齐方式: 把 16 x 16 coarse grid 每个 cell repeat 到 2 x 2，对齐成 32 x 32，再和 fine patch embedding 在 channel 维度 concatenate。
# image size = 512, p_coarse = 32, p_fine = 16
coarse_grid = Trunk1CoarseEncoder(image)       # [B, 16, 16, Dc]
fine_grid   = FinePatchEmbed(image)            # [B, 32, 32, Df]

coarse_up   = repeat_2x2(coarse_grid)          # [B, 32, 32, Dc]
image_grid0 = concat([fine_grid, coarse_up], dim=-1)
image_tokens0 = Linear(image_grid0)            # [B, 1024, D]

# Trunk 3 only needs to solve:
# image_tokens0 + prompt_tokens + K registers -> K prompt-aware visual tokens

如果第一次 encoder 输出不是 spatial grid，而是现有 PrefixMAE 的 abstract 1D tokens，也仍然可以做类似对齐：用 coarse patch stream 的中间输出做 spatial source；或者让 fine tokens cross-attend abstract tokens 得到一个 broadcast memory。前者更简单，建议把第一次 encoder 的 coarse patch tokens 暴露出来。

2. 三类 token 的角色

Token type 输入规模 应该做什么 不应该做什么
image_tokens 1024 fine grid tokens 通过 self-attention 或 local/window attention 看周围 patches，建立空间上下文 不能只被动作为 K/V，否则 patch 之间无法交换证据
prompt_tokens 通常 5 到 128 tokens 形成 word-to-patch relevance，比如“cat”对应图里的猫 patches 不能只做一个 pooled vector，否则会丢掉词级定位能力
register_tokens K 小于 1024，建议 128/256/512 作为最终 1D bottleneck，从 prompt-relevant image tokens 中抽取答案需要的 visual features 不能完全替代 image self-attention，否则只是无上下文 pooling

3. 五种架构设计

方案 1: Fused-Patch + Prompt-Biased Register Perceiver

最推荐 first implementation two streams: image + tokens explicit word-patch bias K registers output

直觉。 先把 coarse high-level feature 和 fine low-level patch embedding 拼成一个 image stream；再让 prompt 生成一张 word-to-patch relevance map；最后 registers 带着 prompt bias 去读取 image stream。它直接实现你想要的效果：prompt 里的“cat”应该在 attention 上激活图像里猫对应的 patches。

I = image_tokens0                                  # [B, 1024, D]
P = prompt_features                                # [B, Lp, D]
R = learned_registers                              # [B, K, D]

for block in Trunk3:
    # 1. image patches see other image patches
    I = I + ImageSelfAttn(LN(I))                    # local/global/window attention
    I = I + ImageFFN(LN(I))

    # 2. prompt reads image and builds word-to-patch map
    P = P + CrossAttn(LN(P), KV=I)                  # attention A_prompt_patch: [Lp, 1024]
    P = P + TextFFN(LN(P))

    # 3. registers read prompt, then read image with prompt-derived bias
    R = R + CrossAttn(LN(R), KV=P)
    bias = make_patch_bias(R, P, A_prompt_patch)    # [K, 1024]
    R = R + CrossAttn(LN(R), KV=I, attn_bias=bias)
    R = R + RegisterFFN(LN(R))

img_features_2 = R                                 # [B, K, D]

关键机制。 A_prompt_patch 是词到 patch 的显式注意力图。registers 不是盲目 attend 所有 patches，而是使用 prompt-derived attention bias。比如某个 register 先 attend 到 prompt token “cat”，它随后读取 image patches 时，attention logits 会被“cat -> image patches”的热图增强。

为什么比普通 Q-Former 更好。 普通 Q-Former 只有 R -> image；这里有 P -> image 先定位，再由 R -> image 抽取。这让 prompt 里的词真正参与决定哪些 patches 被压缩进 1D features。

实现建议。 第一版可以让 ImageSelfAttn 使用 window attention + 每隔几层 global attention，避免 1024 token full attention 过贵。K 从 256 开始，和当前 PrefixMAE 的 num_learnable_tokens 对齐。

方案 2: SAM-R, SAM-style Two-Way Decoder with Registers

SAM-inspired prompt-image two-way image self-attn added interpretable

直觉。 SAM 的 mask decoder 不是单向 cross-attention，而是 two-way transformer：tokens 读 image，image 也反向读 tokens。我们把 SAM 的 mask tokens 换成 VLM registers，把 sparse prompt embeddings 换成文本 prompt tokens，把 mask output 换成 img_features_2。

I = fused_image_tokens                              # [B, 1024, D]
T = concat([registers, prompt_tokens], axis=1)      # [B, K + Lp, D]

for block in SAM_R_Block:
    # token side: registers and prompt exchange information
    T = T + TokenSelfAttn(LN(T))

    # token -> image: prompt/registers look into patches
    T = T + CrossAttn(LN(T), KV=I)
    T = T + TokenFFN(LN(T))

    # image side must also be contextualized
    I = I + ImageSelfAttn(LN(I))                    # local/window or full
    I = I + ImageFFN(LN(I))

    # image -> token: patches are updated by prompt/register information
    I = I + CrossAttn(LN(I), KV=T)

img_features_2 = T[:, :K, :]                        # register slice only

为什么符合你的疑问。 这个结构里 image patches 至少经过 self-attention，而且 prompt/registers 和 image patches 是双向更新。prompt 不只是读取图像，图像 token 自身也被 prompt 语义改写。因此如果 prompt 里有“cat”，猫附近 patches 会在 image stream 中变得更适合后续 registers 提取。

和 SAM 的映射。 SAM 的 mask_tokens 类似这里的 registers；SAM 的 sparse prompt tokens 类似 text prompt tokens；SAM 的 image embedding 类似 fused image tokens。不同点是我们最终保留 registers 作为 VLM 的 1D visual tokens，而不是生成 mask。

可视化收益。 可以额外拿 prompt-to-image attention 和 register-to-image attention 画 heatmap，检查“cat/color/text”等词是否真的落在正确区域。

方案 3: Prompt-AdaLN Fine ViT with Registers

vision-native AdaLN / FiLM conditioning registers in self-attn clean second encoder

直觉。 你提到 AdaLN conditioning 很对，因为它是一种更“vision”的 prompt 注入方式。不要每层都让 prompt cross-attend image，而是让 prompt 生成每层 normalization 的 scale/shift/gate，使整个 fine ViT 变成 prompt-conditioned vision encoder。

I = fused_image_tokens                              # [B, 1024, D]
R = learned_registers                              # [B, K, D]
P = prompt_features                                # [B, Lp, D]

prompt_global = PromptPooler(P)                    # [B, D]

X = concat([R, I], axis=1)                          # [B, K + 1024, D]
for layer in PromptAdaLNViT:
    gamma, beta, gate = AdaLNParams(prompt_global, layer_id)

    X_norm = LN(X) * (1 + gamma) + beta
    X = X + gate.attn * SelfAttn(X_norm)             # registers and patches interact

    X_norm = LN(X) * (1 + gamma.ffn) + beta.ffn
    X = X + gate.ffn * FFN(X_norm)

    if layer_id % m == 0:
        # optional word-level refresh for interpretability
        R = X[:, :K, :]
        R = R + CrossAttn(LN(R), KV=P)
        X = concat([R, X[:, K:, :]], axis=1)

img_features_2 = X[:, :K, :]

本质区别。 方案 1 和 2 都是显式 cross-attention 系统；方案 3 是 prompt-conditioned vision transformer。它更像 DiT/modern diffusion transformer 的 conditioning：prompt 改变每层视觉计算的动态，而不是只作为外部 memory 被读取。

优点。 架构直觉很强：第二次看图就是一个真正的 prompt-conditioned fine vision encoder。image patches 通过 self-attention 自然看其他 patches；registers 作为 prefix tokens 在同一个 self-attention 中逐层吸收视觉信息。

风险。 只用 pooled prompt 会弱化词级定位。因此建议每隔 m 层给 registers 做一次 R -> prompt_tokens cross-attention，或者把 noun/color/OCR tokens 单独保留下来。

方案 4: Explicit Word-Patch Router + Register Pooler

most explicit localization word-to-patch maps router then pool good diagnostic value

直觉。 把“prompt 里的猫 attend 到图像里的猫”做成架构里的显式中间变量，而不是希望 transformer 自己学出来。先计算每个 prompt token 到每个 image patch 的 affinity，然后用这些 heatmaps 指导 registers 做 pooling。

I = ImageContextualizer(fused_image_tokens)         # [B, 1024, D], has self-attn
P = PromptContextualizer(prompt_features)           # [B, Lp, D]

# word-to-patch routing table
A = softmax((P Wq) @ (I Wk).T / sqrt(d), dim=patch) # [B, Lp, 1024]

# create one or more registers per important prompt token
S = TokenSelector(P)                                # select nouns/colors/OCR words or soft gates
R0 = RegisterInit(P, S)                             # [B, K, D]

for block in RouterPooler:
    token_mix = softmax((R0 Wr) @ (P Wp).T)          # [B, K, Lp]
    patch_bias = token_mix @ log(A + eps)            # [B, K, 1024]
    R0 = R0 + CrossAttn(LN(R0), KV=I, attn_bias=patch_bias)
    R0 = R0 + RegisterSelfAttn(LN(R0))
    R0 = R0 + FFN(LN(R0))

img_features_2 = R0

为什么它本质不同。 这里把 localization 和 representation 拆开。先产生可解释的 word-patch maps，再用 maps 约束 register pooling。它不像方案 1 那样把 routing 当成 cross-attention 的副产品，而是把 routing table 当成一等公民。

训练辅助。 可以加轻量正则：heatmap entropy 不要过高，多个关键词的 heatmap 不要完全重合；如果有 OCR/grounding 数据，可以直接监督部分 word-patch maps。

风险。 显式 router 可能过早离散化，导致训练不稳。第一版建议 soft routing，不做 hard top-k。

方案 5: Hierarchical Coarse-to-Fine Register Encoder

coarse-to-fine efficient long sequence soft glimpses OCR and small objects

直觉。 第一次 coarse pass 不只是 high-level feature source，也可以作为第二次 fine pass 的索引结构。prompt 先在 16 x 16 coarse grid 上找大概区域，再只对相关 coarse cells 下的 fine patches 做更重的 attention。

C = coarse_grid                                     # [B, 256, Dc]
F = fine_patch_grid                                 # [B, 1024, Df]
P = prompt_features

# coarse routing: which 32x32 cells are worth expanding?
C_ctx = CoarseSelfAttn(C)
P_ctx = P + CrossAttn(P, KV=C_ctx)
coarse_score = Router(P_ctx, C_ctx)                 # [B, Lp, 256]

# soft expand from each coarse cell to its 2x2 fine patches
selected_fine = SoftGatherFinePatches(F, coarse_score)

# local registers per region, then global registers
R_local = LocalRegisterPooler(selected_fine, P_ctx)
R_global = GlobalRegisterMixer(R_local, P_ctx)

img_features_2 = R_global                           # [B, K, D]

为什么重要。 当第二次 patch size 很小，image sequence 很长时，full attention 不一定划算。这个方案把“粗看一眼”变成计算策略：先粗定位，再细读取。对于文字、小物体、图表局部区域，它可能比对 1024 tokens 做均匀 attention 更高效。

soft 版本。 不要一开始就 hard crop。先做 soft gather 或 coarse-cell weighted fine pooling，等模型稳定后再尝试 top-k cells / ROIAlign。

风险。 如果 coarse router 错过区域，fine encoder 没法补救。可以保留少量 global registers 读取全图 fine tokens，作为 fallback。

5. 推荐实验路线

最建议先做方案 1。 它吸收了你的 channel-concat 想法，又明确解决三者 attention 问题：image patches 先 self-attend；prompt 先生成 word-patch relevance；registers 再用 prompt-derived bias 压缩成 1D features。它比 SAM-R 更简单，比 AdaLN 更容易解释词级 attention，比 hierarchical 方案变量更少。
  1. 先做公共前处理: coarse patch size 32，fine patch size 16，把 coarse feature repeat/interpolate 到 fine grid 后 channel concat。
  2. 实现方案 1: ImageSelfAttn -> PromptCrossAttnToImage -> RegisterCrossAttnToPrompt -> RegisterCrossAttnToImageWithBias。
  3. 保留 ablations: 去掉 prompt bias；去掉 image self-attn；去掉 coarse feature concat；把 K 从 128/256/512 sweep。
  4. 第二个实现做 SAM-R: 如果方案 1 有收益，SAM-R 是最自然的 stronger version，因为它让 image patches 也反向吸收 prompt/register 信息。
  5. AdaLN 作为第三个方向: 如果想验证“更 vision-native 的 prompt conditioning”，方案 3 是很好的替代路径。

7. SAM 架构说明和如何迁移到 VLM

你提到我之前的第三个设计不够清楚。这里单独解释 SAM 的相关部分。SAM 高层有三块：image encoder、prompt encoder、mask decoder。真正对我们有用的是 mask decoder 里的 two-way transformer。

SAM 的关键结构。 Image encoder 先把图像变成 dense image embedding。Prompt encoder 把点、框、mask 等 prompt 变成 sparse prompt embeddings 和 dense prompt embeddings。Mask decoder 把 iou token + mask tokens + sparse prompt tokens 当作 token sequence，把 image embedding 加上 dense prompt embedding 当作 image sequence，然后用 two-way transformer 交互。最后 mask tokens 通过 hypernetwork 作用到 upscaled image embedding 上生成 masks。

图示: SAM 的 two-way decoder 可直接迁移为 VLM 的 prompt-image-register refiner。区别是 SAM 产出 mask，我们产出 1D visual registers。

SAM block 的精确顺序

根据 SAM 官方实现，TwoWayAttentionBlock 每层做四件事：token self-attention；tokens cross-attend image；token MLP；image cross-attend tokens。TwoWayTransformer 最后还会再做一次 token-to-image attention。官方 mask decoder 还会把 iou_token + mask_tokens 和 sparse prompts 拼成 token sequence，再把 dense prompt embedding 加到 image embedding 上。

# SAM-like block, simplified
queries = prompt_and_output_tokens
keys    = image_embedding

queries = SelfAttn(queries)
queries = CrossAttn(q=queries, k=image, v=image)
queries = MLP(queries)
image   = CrossAttn(q=image, k=queries, v=queries)

# final token-to-image attention after all blocks
queries = CrossAttn(q=queries, k=image, v=image)

迁移到这里时需要改的地方

  • Output tokens: SAM 的 mask tokens 换成 VLM registers，数量就是 K，决定 img_features_2 长度。
  • Sparse prompts: SAM 的点/框 prompt 换成 text prompt tokens，最好用 lower-LLM prompt features。
  • Image embedding: SAM 的 dense image embedding 换成 concat(fine_patch_embed, coarse_up) 后的 fused image tokens。
  • Mask head: 不需要生成 mask，但可以保留一个 optional heatmap head，作为解释模型第二次看哪里。
  • Image self-attn: 原始 SAM decoder 对 image token 的 self-attention 很轻；我们这里更需要 image patches 看到其他 patches，所以应显式加入 image self-attn 或 window self-attn。

8. 和现有代码的落地接口

当前 models/siglip_enc_dec.py 里的 ImageEncoder 已经有 patch embedding、patch self-attn、cross-attn 到 learnable tokens。为了支持这份 report 的设计，建议新增三个接口：

  1. encode_coarse(image, patch_size=32, return_grid=True): 返回 16 x 16 coarse grid，而不是只返回 abstract tokens。
  2. fine_patch_embed(image, patch_size=16): 返回 32 x 32 fine patch embedding。
  3. encode_prompt_aware_image(coarse_grid, fine_grid, prompt_features, K): 实现上面五个方案之一，返回 img_features_2: [B, K, D_lm]。

在 models/paligemma_enc_dec.py 中，LLM 输入可以保持四 trunk 顺序：

seq = concat([
    img_features_1,      # first coarse / problem-blind image feature
    prompt_features,     # prompt trunk
    img_features_2,      # prompt-aware second image registers
    answer_features,     # teacher-forced answer trunk
], axis=1)

mask = build_4trunk_mask(len_img1, len_prompt, len_img2, len_answer)

9. Final recommendation

我会把方案 1 作为主线，方案 2 作为 stronger SAM-style 版本。 方案 1 最干净地回答你现在的问题：prompt、image patches、registers 三者如何 attention。它的 block 结构也最容易 ablate。方案 2 则把 SAM 的 two-way intuition 引入进来，可以作为“更强但更重”的第二版。

具体第一版可以这样定义：

Trunk 1:
  image 512, patch 32 -> coarse_grid [16,16,Dc]
  optional Q-Former1 -> img_features_1 [K1,Dlm]

Trunk 2:
  prompt ids -> lower frozen LLM -> prompt_features [Lp,Dlm]

Trunk 3:
  fine patch embed, patch 16 -> fine_grid [32,32,Df]
  coarse_up = repeat_2x2(coarse_grid)
  I0 = Linear(concat([fine_grid, coarse_up], channel))
  I = ImageSelfAttn(I0)
  P = PromptCrossAttn(P, I)               # word-to-patch map
  R = RegisterCrossAttn(R, P)
  R = RegisterCrossAttn(R, I, prompt_bias)
  img_features_2 = R                      # K2 <= 1024

Trunk 4:
  frozen LLM over [img1 | prompt | img2 | answer]
