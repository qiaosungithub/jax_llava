Vision-Centric VLM Understanding-Style Training: 架构建议报告

面向你当前实现（1D visual tokens + LM 中间层分界 + NTP 反传给 vision encoder）的策略建议。 核心目标：让视觉表征更强，而不是让 LLM 接管主要能力。

Executive Summary

先说结论

  1. 你现在的 split 思路是对的：把 LM 前几层当 text feature extractor、后几层当 multimodal fusion + language head，符合 vision-centric 目标。
  2. 仅靠 NTP 容易出现“语言指标变化大，KNN 变化小”的现象，这通常说明视觉表征收益被语言建模噪声掩盖。
  3. 冻结 LM 前半部分一般推荐，但不建议“一刀切长期冻结”；建议用 staged unfreeze（后层先训，前层晚些小 LR 解冻）。
  4. LM 必须预训练加载（至少后半层 + embedding + lm head），否则训练会退化成“同时学语言和视觉”，视觉研究信号不干净。
  5. 如果你要衡量 vision encoder，本质上要引入“弱语言依赖”指标：线性探测、检索、密集预测、跨域稳健性，而不只看 ImageNet KNN。

Current Design Reading

你当前代码设计的优点与风险

优点

  • txt_feature_layer 把文本早层和后层切开，结构上已经接近 dual-encoder + late fusion 的可解释形态。
  • 可选 freeze_lm 与部分层冻结（embedding + layer_0..N-1）已经具备做清晰消融的能力。
  • 1D learnable visual token + cross-attn 压缩，有利于统一上下游输入长度、提高吞吐。

风险

  • NTP 单目标会鼓励“最省力路径”：LM 后层靠语言先验完成 token 预测，视觉分支只需提供弱增益。
  • 冻结后文本任务下降而 KNN 不变，是典型信号：融合层适配变弱，但视觉 backbone 判别能力没有显著改变。
  • 固定长度 K 个视觉 token 可能成为瓶颈：高细节 OCR/grounding 任务容易被压缩损失拖累。

Q2

是否推荐 freeze 前半部分 LM

推荐，但建议“分阶段”而不是永久冻结。

阶段 A（稳定对齐）

  • 冻结 embedding + 前 N 层（N 可取 1/2~2/3 总层数）
  • 训练 vision encoder + projector + LM 后层
  • 目标：先建立视觉到语言空间的稳定接口

阶段 B（温和解冻）

  • 解冻前层中的最后 2-4 层
  • 给这些层较小 LR（后层的 0.1-0.3 倍）
  • 目标：修复冻结导致的语义错配和生成劣化

阶段 C（可选）

  • 若文本指标仍显著掉，再逐步解冻更早层
  • 或仅上 LoRA/adapter，而非全参解冻
  • 目标：最小化 LM 漂移同时补偿性能

你观察到的现象（freeze 后 VQA/文本输出变差，KNN 接近不变）并不意外，通常不代表视觉更差，而是 multimodal 读出头可塑性不足。

Q3

LM 是否必须全部加载 pretrained

建议

  • 建议全部加载预训练权重。 即使部分层后续冻结，也先加载。
  • 不建议把 LM 某些层随机初始化再训，这会把大量训练预算浪费在语言重建。
  • 如果你非常强调 vision-centric，可减少 LM 参数更新量，而不是减少 LM 预训练初始化覆盖率。

更激进但稳妥的变体

  • LM 全预训练 + 大部分冻结 + 小型 adapter/LoRA 仅放在 fusion 相关层。
  • 语言侧做蒸馏约束（保持 text-only perplexity 不漂移），让训练信号更多压到视觉侧。
Q4

如何“只衡量” vision encoder 能力

ImageNet KNN 很有用，但不够。建议至少补齐下面四组：

维度 建议指标 语言依赖 价值
全局语义 ImageNet / iNat / Places 的 linear probe 低 比 KNN 更稳定，能看可线性分离性
检索对齐 COCO/Flickr R@1/5/10（冻结文本编码器） 中 衡量图文匹配质量而非生成技巧
密集感知 VOC/COCO 线性分割头、RefCOCOg IoU 低~中 检验空间结构是否保留
鲁棒性 ImageNet-C / ObjectNet / VTAB transfer 低 检测是否过拟合数据分布

针对你当前现象的解释

  • KNN 接近不变：视觉表征主干未明显受损。
  • VQA/文本变差：LM 后层融合和生成适配受限，属于“读出层问题”多于“视觉骨干问题”。
  • 建议加一个“冻结 LM 的 image-to-text retrieval”指标，能更直接看视觉贡献。
Q5

架构改进建议（按优先级）

P0: 保持框架不大改、最快见效

  • 分组 LR：vision/projector 高 LR，LM 后层 低 LR，LM 前层 冻结或超低 LR。
  • 加 token-budget curriculum：先大 K（易学），逐步增加 mask/drop，逼迫 token 效率。
  • 在 NTP 外加轻量对比损失（image-text symmetric contrastive），权重小但持续存在。

P1: 中等改动

  • 把单次 concat 改为“多层注入”：在 LM 后几层重复 small cross-attn 或 FiLM/gated adapter。
  • 对视觉 token 做层次化：global tokens + local tokens，分别服务语义与定位。
  • 在 projector 前后加 residual gating，控制“视觉信息注入强度”。

P2: 更激进方向

  • 双塔训练头（CLIP-style）与生成头共存：共享 vision encoder，联合优化检索与生成。
  • MoE visual tokens 或动态 token pruning，让模型按样本复杂度分配视觉预算。
MAE Integration

能否自然引入 MAE pretraining

可以，而且不“投敌”。 关键是把 MAE 作为视觉表征正则，而不是终点目标。

  • 方案 1：两阶段。先短期 MAE 预热 vision encoder，再切到 understanding-style 多模态训练。
  • 方案 2：联合训练。保留小权重 recon_loss_weight，但随训练衰减到接近 0。
  • 方案 3：只对早层施加重建（patch-level），后层主要受语义损失驱动。

原则：重建任务用于“保结构与低层纹理”，主任务仍由视觉语义与图文对齐决定。

CoCa Lessons

来自 CoCa 的可迁移 Recipe（已对齐你当前设计）

结构层面

  • CoCa 将 decoder 一分为二：前半层做 text-only（unimodal），后半层做 image-text 融合（multimodal）。
  • 这和你的 txt_feature_layer 思路一致；他们的结论也是“前后分治”比纯早融合或纯晚融合更平衡。
  • CoCa 小规模消融显示 n_uni:n_multi 取约 1:1 是稳定点，你可优先扫 1/3、1/2、2/3 三档。

损失与权重

  • CoCa 核心是联合目标：L = λ_cap * L_cap + λ_con * L_con，不是只靠 NTP。
  • 他们的 ablation 倾向 λ_cap:λ_con = 2:1 优于 1:1/1:2。
  • 迁移到你这里：建议保持 VLM/NTP 为主，但持续保留小权重对比损失，避免视觉表征被语言先验“吞掉”。

视觉 token 与 pooler

  • CoCa 用长视觉序列服务生成分支（等价于“保细粒度视觉信息”），而对比分支用更紧凑的全局 embedding。
  • 对你而言：保留较大的 K（如 256）更利于 OCR/grounding；过早压缩 K 往往只会让文本任务先掉。
  • 如果做对比头，优先用简单 text head（单 CLS / 单 query）减少对比与生成目标冲突。

Practical Defaults

一个可直接开跑的默认配方

LM: 全量加载 pretrained txt_feature_layer: 9 或 12 阶段A冻结前层 vision LR = LM 后层 LR 的 3x K = 256 起步 小权重 contrastive + NTP recon_loss 前高后低衰减
  • 若你首要目标是视觉表征提升，优先看：linear probe / retrieval / refcocog，而不是先盯 VQA 生成分数。
  • 当这些视觉指标稳定提升后，再做少量 LM 解冻去回收文本输出性能。
